## 0. 발표 한 줄 컨셉
**"받을 수 있는 지원이 아니라, 지금 받아야 하는 지원을 알려줍니다."** — 시간축 내비게이션으로 임산부의 *놓치는 돈*을 막는다.

## 2. 슬라이드별 상세

### S1 — Answer First (`hero`)
- **eyebrow**: 임산부를 위한 지원 내비게이션
- **title**: 받을 수 있는 지원이 아니라, **지금 받아야 하는** 지원
- **subtitle**: 임신 주차·예정일·거주지를 입력하면 "지금 신청 → 곧 열림" 시간순 정렬 + 마감 D-day 경고
- **footnote**: 4시간 AI 해커톤 / 데모 지역·제도 한정

### S2 — Problem (`problem-flow`)
- **title**: 정보가 없어서가 아니다
- **items(흐름)**:
  1. 박지은, 32세, 서울, 첫 임신 8주차, 맞벌이 — 입덧·야근에 치여 알아볼 여유가 없다.
  2. 채널이 부처별로 갈라짐(행안부/복지부) → 전체 그림이 안 보임.
  3. 전부 pull 방식 → "언제 움직여야 하는지" 아무도 먼저 안 알려줌.
  4. **신청 창을 놓치면 돈이 사라진다** — 부모급여·첫만남이용권은 **출생일로부터 60일 이내** 신청해야 출생월부터 소급(검증완료).

### S3 — Insight: 통념↔현실 (`contrast`)
- **title**: 정보량과 수혜는 비례하지 않는다
- **leftLabel / left (통념)**: 못 받는 건 정보가 부족해서다 → 모아주면 된다.
- **rightLabel / right (관찰된 현실)**: 정보는 차고 넘치는데도, 신청 시기·소급 마감을 놓쳐 받을 수 있던 돈을 잃는다.
- **lead(긴장)**: "무엇이 있는지"를 알아도 "언제 움직일지"를 모르면 지원은 통째로 사라진다.

### S4 — Insight 리프레이밍 (`insight-statement`)
- **title**: 이건 정보 부족이 아니라 **타이밍 문제**다
- **subtitle**: 임산부에게 필요한 건 카탈로그가 아니라 **"지금 뭘 해야 하는지 알려주는 시계"**.

### S5 — Solution (`product-overview`)
- **title**: 입력 한 번 → 시간축 한 화면
- **features**:
  - **입력**: 출산예정일(또는 임신 주차) + 거주지(시/도·시군구).
  - **시간축 정렬**: 입력을 D-day·아기 월령으로 정규화 → 각 제도 window와 매칭해 `now`/`soon`/`past` 분류.
  - **손실 경고**: 소급 마감 임박 제도를 D-day 손실 프레임으로 점등.
  - **AI 브리핑(P2)**: 비정형 교차 질의("쌍둥이+고위험+전입 3개월")를 검증 데이터 안에서만 추려 답(환각 차단). 폴백=규칙 템플릿.

### S6 — Demo / Wow Moment (`demo-callout`) ★핵심
- **title**: 입력을 바꾸면, 화면이 살아 움직인다
- **callout(placeholder)**:
  `[데모: 임신 주차 슬라이더를 출산 직후로 당기면 → 타임라인 카드가 눈앞에서 재정렬되고, 첫만남이용권·부모급여 카드에 "D-N 지나면 소급분 소멸" 손실 경고가 빨갛게 점등]`
- **points**:
  - 입력 변경 전후 "지금 받을 수 있어요" 그룹의 카드 구성/순서가 바뀐다.
  - 마감 임박 카드에 **손실 경고 배지**가 점등(`D-`·`소멸/마감`).
  - (P2) 자유질문 → AI가 데이터 근거 제도만 추림.
- ⚠️ **실제 화면 아직 없음.** 캡처는 Stage 07 데모검증 후 교체. 지금은 placeholder로만 표기, 완성처럼 단언 금지.

### S7 — Mechanism / 벤치마킹 이식 (`architecture`)
- **title**: 검증된 해외 메커니즘을 한국 제도에 이식
- **steps**:
  1. 영국 GOV.UK **Sure Start** — 클레임 윈도우(기한을 1급 정보로).
  2. 호주 **Services Australia** — 출산 전 사전신청(시간축 분리).
  3. 에스토니아 **proactive** — 능동성(먼저 알려줌).
  4. **mRelief** 자격 스크리너 — 최소질문 자격 추림의 LLM 업그레이드.
- **result**: 기능 통째가 아니라 **메커니즘만** 축소 이식 — "기한을 1급 정보로 + 시간축 분리 + 최소질문 자격 추림".

### S8 — Credibility / 차별화 (`contrast`)
- **title**: 정부24가 이미 통합 안내하지 않나?
- **leftLabel / left**: 정부24 = 신청 **창구** (무엇을 신청할지는 사용자가 알아야 함).
- **rightLabel / right**: 우리 = **언제·어디로** 갈지 판단하는 시간축 레이어.
- **lead**: 데모의 라이브 재정렬+마감 경고로 그 차이를 30초 안에 보인다.

### S9 — Impact (`card-grid` 또는 `before-after`)
- **title**: 무엇이 바뀌나 (방향성)
- **cards**:
  - **누락 가능성 감소** — 시간순 정렬로 놓치는 제도↓.
  - **의사결정 시간 단축** — 여러 포털 탐색 → 한 화면.
  - **마감 손실 회피** — 소급 마감 경고로 사라질 돈을 막음.
- **검증수치(단언 가능, verified-figures.md 출처)**: 부모급여·첫만남 **60일 소급 마감**, 첫만남 첫째 200/둘째이상 300만원, 부모급여 0세 월 100/1세 월 50만원, 산모·신생아 **출산예정 40일 전~출산 후 30일** 창.
  ⚠️ **임의 환산·효과 % 만들지 말 것.** "X% 절감" 같은 미검증 정량은 금지.

### S10 — Closing (`closing`)
- **title**: 받을 수 있는 지원이 아니라, **지금 받아야 하는 지원**을 알려줍니다.
- **subtitle**: 임신은 타이밍 게임 — 우리는 그 시계가 됩니다.
- **cta**: 공식 신청처로 안전하게 연결(실제 신청·인증·결제는 하지 않음).
- (closing 문구는 concept.md 마지막 문장 그대로 — 변경 금지.)

## 3. 발표에서 단언 가능한 검증 수치 (verified-figures.md only)

- **부모급여·첫만남이용권 60일 소급 규칙** ⭐: 출생일로부터 **60일 이내** 신청 시 출생월부터 소급, 초과 시 신청월부터만 지급(그 사이 월분 손실). [verified: true]
- **첫만남이용권**: 첫째 200만원 / 둘째 이상 300만원(2024.1.1 이후 출생), 사용기한 생년월일+2년. [true]
- **부모급여(2026)**: 0세 월 100만원 / 1세 월 50만원. [true]
- **아동수당** ⚠️정정: 월 10만원, **2026년 9세 미만(0~107개월)으로 확대** + 비수도권/인구감소지역 최대 월 3만원 추가. ("8세 미만" 금지) [true·정정]
- **임신·출산 진료비 바우처** ⚠️정정: 일태아 100만원 / **다태아 140만원 기본 + 태아당 100만원 되도록 추가(쌍둥이 ≈200만원)**, 사용기한 분만예정일+2년. ("다태아 정액 140" 금지) [true·정정]
- **산모·신생아 건강관리**: 신청 **출산예정 40일 전~출산 후 30일**, 기준중위소득 150% 이하. [true]
- **고위험 임산부 의료비**: 2024년부터 소득기준 폐지, 19대 질환, 90% 한도 300만원, 분만일로부터 6개월 이내. [true]
