## 1. 발표 흐름 (슬라이드 단위 초안)

### S1. 표지
- **제목:** 우리 가족 맞춤 정부 혜택 찾기 — "왜 당신인지"를 보여주는 AI 복지 안내
- **핵심 메시지:** 가족 상황을 한 줄로 말하면, 로그인 없이 30초 만에 받을 수 있는 혜택과 "왜 당신이 해당되는지"를 AI가 설명한다.
- (근거: concept.md 한 문장 피치)

### S2. Problem — 신청주의의 사각지대
- **제목:** 받을 수 있는데 못 받는다
- **핵심 메시지:** 한국 복지는 신청주의 — 제도가 많아도(복지멤버십 안내 대상만 128종, 중앙·지자체·민간 분산) 내 가구 맞춤 전체를 못 본다. 정확한 안내는 로그인·본인인증·정보연계 동의 뒤에 있어 진입에서 이탈하고, "기준이 엄격해 난 안 될 것 같다"는 자기검열로 시도조차 안 한다.
- **페르소나 1줄:** 8개월 아기 키우는 외벌이 부부 박지은(33), 서울 — "우리가 받을 수 있는 게 있긴 할까?"
- (근거: research §1, spec §2 / `[jtbd-2]`, `[dom-2,3]`, `[jtbd-3, unverified]`)

### S3. Insight — "무엇을"이 아니라 "왜 당신이"
- **제목:** 목록은 넘치는데, 왜 안 받을까
- **핵심 메시지:** 정보·목록은 이미 넘치는데 행동으로 안 이어진다. 복지 안내의 핵심은 "무엇을 받을 수 있나"가 아니라 "왜 당신이 해당되는가"다. 그리고 AI의 자리는 자격 판정이 아니라 — 막연한 말을 구조화하고 규칙의 결과를 사람 말로 풀어주는 "통역사"다.
- (근거: spec §3 Reframing)

### S4. Solution / 데모 진입
- **제목:** 한 줄 입력 → AI 구조화 → 규칙 매칭 → "왜?" 증명
- **핵심 메시지:** 자연어 한 줄 입력(로그인 없음) → AI가 가구유형·소득구간·지역으로 구조화 → 규칙 엔진이 결정론적으로 매칭(판정은 LLM이 아님) → 혜택 카드마다 ✓충족 체크리스트 + "왜?" 배지 + AI가 다듬은 한 줄 설명 + 신청 동선.
- (근거: concept.md 킥, spec §7)

### S5. Wow Moment (데모)
- **제목:** 로그인 없이, 30초 만에 "왜 당신인지"
- **핵심 메시지:** [여기서 라이브 데모 / 캡처] — 자연어 한 줄을 입력하자마자 로그인 화면을 한 번도 거치지 않고 혜택 카드에 ✓충족 체크리스트와 "왜 당신이 해당되는지"가 즉시 뜬다.
- (근거: concept.md Wow / spec §9. 데모 placeholder는 §3 참조)

### S6. Differentiation — 복지로 대비
- **제목:** 이미 있다고요? 결정적 차이는 두 가지
- **핵심 메시지:** 기존(복지로·보조금24)은 로그인·정보연계 동의가 전제이고 결과는 "왜"가 없는 목록이다. 우리는 (1) 로그인 0회로 (2) "왜 당신이 해당되는지"까지 보여준다. 차별화 수준은 Insight + 시연 차별화 — 관점과 무로그인 즉시 체험이 데모로 증명된다.
- **벤치마킹(메커니즘만 이식, 규모는 안 베낌):** rules-as-code = ACCESS NYC·mRelief / "왜 해당되나" 근거 가시화 = Turn2us / 익명·무저장·"보장 아님" = USA.gov Benefit Finder.
- (근거: spec §6 / `[dom-2,3]`, `[ovs-1,3,4,6]`)

### S7. Impact (방향성으로만)
- **제목:** 마찰을 없애고 확신을 더한다
- **핵심 메시지:** 혜택 발견까지의 단계 감소(로그인·정보연계 동의 제거 → 진입 마찰↓), "왜 해당되는지" 근거 제공으로 신청 결정 확신↑·자기검열 누락 가능성↓, 자연어 입력으로 개인화 안내 진입 속도↑.
- ⚠️ 구체 수치 없음 — 방향성으로만. (근거: spec §10)

### S8. Credibility — 가장 강한 의심 방어
- **제목:** "그냥 검색 + ChatGPT 아닌가요?"
- **핵심 메시지:** ① 제한된 사용 범위 — 자격 판정은 규칙 엔진이 결정론적으로, AI는 확정 텍스트를 다듬을 뿐(사실 생성 금지·폴백). ② 작동 증거 — 설명 옆에 규칙 충족 근거(소득<기준 등)를 나란히 띄워 "지어낸 말"이 아님을 화면으로 증명. ③ 차별 근거 — 기존은 로그인·정보연계 전제, 우리는 로그인 0회 + "왜"까지.
- **선제 방어("로그인 0회는 진짜 데이터 안 써서 가능한 것 아니냐"):** 맞다 — 큐레이션 예시 데이터(fixture 8~10건) 기반의 의도된 데모 선택. 주장은 "전수 안내"가 아니라 "예시 가족 시나리오에서 '왜'를 증명한다".
- (근거: spec §11, §13 / `[dom-2,3]`, `[judge-1,2]`)

### S9. Closing
- **제목:** 정부는 "받을 수 있다"고만 합니다
- **핵심 메시지(마지막 문장 그대로):** 정부는 "받을 수 있다"고만 합니다. 우리는 "왜 당신인지"를 보여줍니다.
- (근거: concept.md / spec §14)

## 2. 시간 배분 (5분 = 300초)

| 슬라이드 | 내용 | 배분 |
|---|---|---|
| S1 | 표지 / 한 문장 피치 | 약 20초 |
| S2 | Problem (신청주의 사각지대) | 약 45초 |
| S3 | Insight ("왜 당신이") | 약 35초 |
| S4 | Solution / 데모 진입 | 약 30초 |
| S5 | **Wow Moment (라이브 데모)** | 약 70초 |
| S6 | Differentiation (복지로 대비) | 약 40초 |
| S7 | Impact | 약 20초 |
| S8 | Credibility (의심 방어) | 약 30초 |
| S9 | Closing | 약 10초 |
| 합계 | | **약 300초 (5분)** |

> 시간 압박 시 우선순위: Wow Moment(S5) > Credibility(S8) > Differentiation(S6) 순으로 사수. Impact(S7)는 한 문장으로 압축 가능.

- **장면:** 첫 화면에서 로그인 없이 자연어 한 줄 입력 — *"30대 부부, 8개월 아기, 서울, 월 350만원"* → '혜택 찾기' 클릭.
