# 우리 동네 맞춤 재난 대비 가이드

> 거주 지역과 가족 구성만 입력하면, **우리 동네·우리 가족 상황에 꼭 맞는 재난 행동요령과 가까운 대피 정보**를 한 화면에서 안내하는 공공 서비스 프로토타입입니다.

2026 AI 해커톤 출품작 · [KRDS(대한민국 디지털정부 디자인시스템)](https://www.krds.go.kr/) 기반

---

## 1. 해결하려는 문제

재난 상황에서 국민이 가장 먼저 찾는 것은 **"지금 내가, 우리 가족이 무엇을 해야 하는가"** 입니다. 그러나 현재 재난 정보는 이렇게 흩어져 있습니다.

- 행동요령은 행정안전부 국민재난안전포털, 대피소는 지자체·안전디딤돌, 무더위쉼터는 또 다른 포털, 실시간 경보는 문자/기상청 등 **출처가 제각각**입니다.
- 정보가 **"평균적인 국민"** 기준이라, 영유아·임산부·노인·장애인·반려동물 등 **내 가족 구성에 맞는 준비물·대피 방법**을 따로 찾아야 합니다.
- 정작 재난이 닥친 순간에는 이 정보들을 일일이 검색할 여유가 없습니다.

**이 서비스는 "지역 + 가족 구성"이라는 두 가지 입력만으로, 흩어진 정보를 한 번에 모아 맞춤형으로 보여줍니다.**

## 2. 주요 기능

| 기능 | 설명 |
|---|---|
| 🏠 **지역 입력** | 시/도·시/군/구를 선택하면 해당 지역의 재난 위험과 대피 정보를 기준으로 안내 |
| 👨‍👩‍👧 **가족 구성 입력** | 영유아·임산부·노인·장애인·반려동물 등 해당 항목 선택 |
| 🚨 **재난 유형별 행동요령** | 지진·호우/침수·폭염·한파·화재·태풍 등 유형별 핵심 행동요령을 **심각도(위험/주의/안내) 배지**와 함께 카드로 제시 |
| 🧑‍🤝‍🧑 **우리 가족 맞춤 안내** | 선택한 가족 구성에 맞춰 비상 준비물·우선 대피·복약·동반 대피소 등 맞춤 팁 제공 |
| 🗺️ **가까운 대피소** | 지도 위에 인근 대피소/쉼터 표시 (지도 연동 영역) |

> 현재 프로토타입은 정적 데이터로 화면 흐름을 구현했으며, 아래 공공데이터 연동을 통해 실데이터로 대체하도록 설계되어 있습니다.

## 3. 기술 스택

- **프레임워크**: Next.js 14 (App Router) + React 18 + TypeScript
- **스타일**: Tailwind CSS 3
- **디자인 시스템**: **KRDS(대한민국 디지털정부 디자인시스템)**
  - 공식 React 컴포넌트 라이브러리 [`@krds-ui/core`](https://github.com/KRDS-community/krds-react) (Apache-2.0) 사용
  - Masthead(정부 식별), 타이포그래피(Display/Heading/Body/Label/Detail), Chip, TextInput, Button, Badge, Link 등 공식 컴포넌트 적용
  - 색상·타이포 등 디자인 토큰은 KRDS 공식 토큰을 그대로 차용 (`web/tailwind.config.ts`)
- **폰트**: Pretendard GOV (KRDS 권장 본문 폰트)

## 4. 활용(예정) 공공데이터

> 원칙: **잘 바뀌지 않는 위치/시설 = 정적 다운로드**, **시시각각 바뀌는 상태/경보 = 실시간 API**. 자세한 목록과 신청 경로는 [`data/data-sources.md`](data/data-sources.md) 참고.

- **행정구역 경계** — 시도/시군구 경계 GeoJSON·TopoJSON (지역 선택·지도 베이스, 이미 확보)
- **무더위/한파쉼터** — 전국무더위쉼터표준데이터
- **대피 시설** — 전국지진옥외대피장소·전국민방위대피시설 표준데이터, 도시공원·공공시설 개방정보
- **어린이보호구역** — 전국어린이보호구역표준데이터 (영유아 가구 안내)
- **안전 인프라** — 전국CCTV·안전비상벨 위치정보
- **기상특보** — 기상청 한파·폭염·호우 등 12종 특보
- **대기오염** — 에어코리아 실시간 미세먼지/PM2.5
- **긴급재난문자** — safetydata.go.kr 실시간 재난문자
- **복지·혜택** — 복지로(복지서비스정보)·보조금24 (가족 구성별 지원 연계)

각 실시간 API는 **호출 실패 시 정적 백업 데이터로 fallback** 하도록 설계해 데모 안정성을 확보합니다.

## 5. 로컬 실행 방법

```bash
cd web

```

브라우저에서 <http://localhost:3000> 접속.


## 6. KRDS 기반 디자인 · 접근성 고려 사항

- **정부 서비스 정체성**: KRDS `Masthead`로 정부 식별 영역(태극기·공식 누리집 안내)을 제공해 공공 서비스 신뢰도를 확보했습니다.
- **일관된 디자인 토큰**: 색상(정부 블루 `#256ef4`), 타이포 스케일, 라운드 등 KRDS 공식 토큰을 사용해 다른 정부 누리집과 시각적 일관성을 유지합니다.
- **명도 대비 / 의미색**: 재난 심각도를 KRDS `Badge`의 위험(danger)·주의(warning)·안내(primary) 색으로 구분해 직관적으로 전달합니다.
- **키보드 · 포커스 접근성**: 모든 인터랙티브 요소에 명확한 포커스 표시(`:focus-visible` 2px 아웃라인)를 적용했습니다.
- **시맨틱 마크업**: `fieldset`/`legend`로 폼 그룹을 구조화하고, 결과 영역에 `aria-live`를 적용해 스크린리더 사용자에게 결과 갱신을 알립니다.
- **반응형**: KRDS 모바일/PC 타이포 스케일을 활용해 모바일에서도 가독성을 유지합니다.

> **참고**: `@krds-ui/core` 0.6.0의 `Select` 컴포넌트는 선택값을 외부로 노출하지 않는 비제어형이라, 폼 상태 연동이 필요한 **시/도 선택**에는 KRDS 토큰을 적용한 native `<select>`를 유지했습니다(KRDS 셋업 가이드 4항). 그 외 입력 요소는 모두 KRDS 공식 컴포넌트를 사용합니다.
