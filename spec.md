# spec.md — 행정 온램프 (가칭)

## 1. Answer First
행정 신고가 낯선 사회초년생을 위해, **받은 무서운/처음인 행정 문서·상황**을 사진·붙여넣기·말로 입력하면 **AI(멀티모달+LLM)가 읽어 "쉬운 말 뜻 + 기한 + 당장 첫 걸음 + 어디로(관할) + 출처"로 바꿔주고**, 실제 처리는 정부24로 넘기는 모바일 도구.

## 2. Problem
- **사용자:** 행정 신고 경험이 거의 없는 청년(확장: 고령자·외국인 주민).
- **사용 상황:** 법원 지급명령·과태료 고지서·공단 통지 등 *낯선 문서*를 받았거나, "이사했는데 뭐부터?" 같은 *처음 닥친 상황*에 직면한 직후.
- **현재 방식:** 네이버·블로그/카페 검색, 정부24 검색, 지인 카톡, 콜센터.
- **현재 방식의 실패:** 블로그는 최신성·정확성 판단 불가, 정부24는 행정 언어라 검색 미스매치(시민 언어와 안 맞음), 결국 "맞는지 몰라" 멈춤. 정부24는 *내가 받은 그 문서*를 번역해주지 않음.
- **실패 비용:** 기한 초과(지급명령 2주 놓치면 확정판결 효력 → 압류), 서류 미비 반려·재방문, 받을 수 있던 지원 누락.

## 3. 최종 Insight
**Common Assumption:** 행정이 어려운 건 정보가 흩어져서다 → 통합하면 된다.
**Observed Reality:** 정부24가 이미 통합·AI챗봇까지 붙였는데도 사람들은 *겁부터 내고* 헤맨다.
**Tension:** 정보는 충분한데 "첫 걸음"을 못 뗀다.
**Reframing:** 행정 접근성은 *정보 부족*이 아니라 **"첫 걸음 앞의 공포·막막함"** 문제다. 필요한 건 더 많은 정보가 아니라 **겁을 없애고 첫 걸음을 떼게 하는 온램프**.

## 4. JTBD
- **Functional:** 받은 문서/상황이 무슨 뜻이고, *지금 당장 무엇을* 어디서 해야 하는지 한 번에 파악한다.
- **Emotional:** "뭔가 놓쳐 불이익 받을까" 하는 **불안을 확신으로**, 창구에서 무지를 들킬 **민망함을 회피**.
- **Social:** "행정 정도는 알아서 처리하는 제 앞가림하는 어른"으로 보이고 싶다.
- **Current Alternative:** 블로그·카페·지인·콜센터·정부24 검색.
- **Success Criteria:** "이 한 화면만 보면 첫 걸음은 떼겠다"는 확신 — 첫 걸음·기한·관할이 콕 집히고 출처가 보임.

## 5. AI Leverage
- **AI 없이도 가능한 것:** 정적 안내 페이지·공식 링크 제공(정부24가 이미 함).
- **AI로 더 빠르고 싸지는 것:** 어려운 용어 풀이, 절차의 단계화.
- **AI 때문에 처음 가능해지는 것:** 사용자가 **받은 임의의 문서(사진 포함)·미리 못 짜둔 상황**을 즉석에서 읽어 *그 문서 고유의* 첫 걸음을 뽑아내는 것. ← 검색·규칙으로 불가. (엔진은 **AI only**: 멀티모달 비전+LLM, 규칙/하드코딩 '가짜 바닥' 없음.)
- **AI가 맡는 역할:** 문서·사진·상황 판독, 쉬운 말 변환, 첫 걸음·관할 추론, **공식 출처 인용**.
- **AI가 하지 않는 역할:** 실제 신청·발급·제출 대행, 확정적 법률 자문, 정확도 보증.
- **오류 가능성:** 환각(틀린 절차/기한), 관할 오판, 사진 오독.
- **사람이 최종 판단:** 실제 제출 전 **공식 출처/관할 창구에서 확인**(가드레일).

## 6. Differentiation
기능 차별화가 아니라 **상위(Insight·Narrative) 차별화**를 노린다.
- **Insight 차별:** 정부24=신청(미는 방향) ↔ 우리=받은 문서(받는 방향) 진입점.
- **사용자 흐름 차별:** "조회"가 아니라 "받은 종이 → 첫 걸음 하나"로 끝까지 손잡음.
- **시연 차별:** 즉석 입력을 라이브로 처리(대본 아님).
- **신뢰 차별:** 답마다 출처 인용 + "모르면 모름"(프랑스 Albert ↔ NYC MyCity 반면교사).
- **Narrative:** "겁 → 첫 걸음".

## 7. Solution
**입력**(받은 문서 사진 / 붙여넣기 / 상황 설명) → **AI 처리**(멀티모달 판독 → 의미·기한·첫 걸음·관할 추론 → 공식 출처 근거 인용, 근거 없으면 '확인 필요') → **출력**(한 화면 4칸 카드 + 출처 배지) → **사용자의 다음 행동**(첫 걸음 실행 후 정부24/관할 창구로 핸드오프).

## 8. 데모 시나리오
가상 페르소나 **김民우(27)**가 **지급명령**을 받은 상황. ① 통합 입력칸에 문구 붙여넣기(또는 사진/'예시로 해보기') → '분석' → ② [뜻·기한 2주·첫 걸음(이의신청)·관할] 4칸 카드 → ③ 각 항목 **출처 배지** → ④ **심사위원이 즉석 입력**한 다른 문서/상황도 같은 엔진이 처리. (상세·selector/assertion: `demo/demo.scenario.yaml`.)
**데모 형태:** 모바일 **PWA**(사진 촬영·홈 화면 추가). 위치/지도(관할 표시)는 스트레치, 알림은 데모 범위 밖. 반응형은 기본 전제.

## 9. Wow Moment
즉석 입력(또는 지급명령 사진/텍스트) → 수 초 뒤 무서운 한자어 문서가 **[뜻·기한·첫걸음·관할·출처] 카드**로 바뀌고 **출처 배지가 강조**된다. 검증: `[data-testid="guide-card"] [data-testid="source-badge"]` 가 함께 렌더(= 쉬운말+행동+신뢰가 한 장면에). "막연한 공포 → 믿을 수 있는 첫 걸음."

## 10. Impact
(근거 없는 수치 금지)
- 첫 행동까지의 **단계·시간 단축 가능성**(블로그 30분 탐색 → 한 화면).
- 기한 인지로 **누락(기한 초과) 가능성 감소**.
- 출처 동반으로 **신뢰 판단 비용 감소**.
- 쉬운 말·즉석 입력으로 **접근 대상 확대 가능성**(고령자·외국인).

## 11. Credibility
가장 강한 의심 = **"정부24/AI정부24와 뭐가 다른가 + 환각으로 틀리면?"**
- **데이터 근거:** 답은 **사전 수집한 소수의 공식 출처 스니펫(URL 포함) fixture에서만** 인용한다(LLM이 출처 URL을 지어내지 않음). 매칭되는 근거가 없으면 §13대로 "공식 확인 필요"로 회피. → 엔진 추론은 AI, *출처 코퍼스는 검증된 fixture*: "AI only"와 충돌 아님(가짜 바닥이 아니라 신뢰 레이어 = 환각 URL 차단).
- **실제 작동 증거:** 즉석 입력을 라이브로 처리해 보임.
- **제한된 범위:** 데모는 지급명령(+전입신고) 중심, "이 범위 안에서는 공식 출처만".
- **사람 검토 지점:** 실제 제출 전 공식 창구 확인 안내.

## 12. Limitation
- **Soft:** 용어 풀이의 뉘앙스·표현이 완벽하지 않을 수 있음.
- **Bounded:** 데모는 일부 민원(지급명령·전입신고)만 깊게. 전 민원 전수 아님(모집단은 넓히되 시연은 일부).
- **Hard:** 법적 효력 있는 자문·실제 신청 대행 불가. 최신 제도 변경의 실시간 반영은 한계 → 출처·기준일로 보완.

## 13. Guardrail
- **잘못된 결과 표시:** 근거 출처가 없으면 단정하지 않고 **"공식 확인 필요 + 링크"**로 회피.
- **최종 책임:** 사용자 본인과 공식 기관(우리는 안내 보조).
- **사용하지 말아야 할 경우:** 긴급·중대한 법적 분쟁은 전문가(변호사)·공식 창구 우선.
- **사람 개입 지점:** 실제 제출·신청 직전 공식 출처/관할 창구 확인.

## 14. Closing Message
**"정부24는 '신청 버튼'을 줍니다. 우리는 그 버튼이 어디 있는지조차 모르는 사람을, 거기까지 데려다줍니다."**

## 15. 범위 밖
- 실제 신청·발급·제출 대행(정부24로 핸드오프).
- 전 민원 전수 시연(데모는 시나리오 중심).
- 알림/푸시, 음성·다국어(언급/확장만), 회원가입·본인인증.
- 관할의 "최근접 거리" 추천(관할은 주소·사건 기준만).

## 16. 4시간 현실성
**AI only라 빌드가 단순**(이중 규칙 경로 없음). 우선순위 빌드 = ① 단일 페이지 + 통합 입력 + `/api/llm`(텍스트) 가이드 카드(핵심) → ②  **출처 스니펫 fixture(지급명령·전입신고 공식 출처 5~15건, URL 포함) 인덱싱 + LLM이 그중에서만 인용**(출처 배지 href는 이 화이트리스트에서만) → ③ 즉석 입력 동작 → ④ 사진=멀티모달 비전 경로(llm.ts 확장) → ⑤ 전입신고 2번째 시나리오 → ⑥ 위치/지도. **①~③까지가 반드시 끝까지 도는 핵심**(②의 출처 fixture는 신뢰 레이어라 핵심에 포함 — 별도 작업으로 견적), ④~⑥은 시간 남으면. Wow 1개(즉석→카드+출처)에 집중. 외부 의존은 OPENAI_API_KEY 한 곳(비전 가능 모델 필요) — 데모 보험은 키 사전준비 + 검증 샘플 + 녹화(코드 폴백 없음).
