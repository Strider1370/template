## 1. 모바일 디자인 체크리스트
**대상: `docs/mobile-design-checklist.md` (신규 파일 — 아래 내용 그대로 생성)**

```markdown
# 모바일 디자인 체크리스트 (실전)

> CLAUDE.md 고정 제약: **모바일 반응형은 모든 주제 필수.** 그런데 "폰 폭에서 안 깨진다 ≠ 모바일 앱"이다.
> 반응형 웹을 그대로 두면 "폰에 욱여넣은 웹사이트"로 보인다. 실제로 데모를 폰/미러링으로 보여줄 때 이 차이가 크게 드러난다.
> 주로 Stage 05(구현)·07(데모검증)에서 확인. 전부 JS/CSS라 빠르게 고칠 수 있다.

## 1. 웹사이트 크롬을 모바일에선 걷어내라 (반응형 분기)
데스크톱 웹의 흔적을 모바일에서 숨기고, **앱 셸**(컴팩트 헤더 + 본문 + 핵심 행동)로 재구성한다. `md:` 분기로 데스크톱은 그대로 보존.
- **상단 GNB·정부 Masthead 같은 큰 헤더** → 모바일 숨김(`hidden md:block`). 앱 헤더는 **로고+이름 1줄**로 충분.
- **큰 히어로 배너** → 모바일에서 패딩·폰트 축소(예: `pb-5 pt-6 md:pb-16`, `text-heading-m md:text-display-m`).
- **핵심 입력/행동이 첫 화면(fold)에 보이게.** 슬로건이 입력보다 위 무게를 차지하면 우선순위가 틀린 것.
- **하단 탭바는 목적지가 있을 때만.** 단일 행동 앱(입력→결과)엔 과함. "최근 내역" 같은 destination이 생기면 그때.

## 2. 한국어·터치·입력 디테일
- **한글 줄바꿈**: 제목에 `break-keep`(word-break: keep-all). 안 주면 "쉬운 첫 걸**음**"처럼 음절 중간에서 깨진다.
- **하단 고정(sticky) CTA는 양날의 검**: 엄지 접근성↑이지만 **결과가 길어지면 스크롤 중 오탭** 위험. 결과형 화면에선 인라인이 안전할 때가 많다(사안별 선택).
- **사진/파일 입력**: 작은 "첨부됨" 배지보다 **입력창(textarea)을 미리보기 블록으로 대체**하는 게 상태가 분명. 빼기 버튼·초기화 버튼 같이.
- **입력 충돌 주의**: 예시 텍스트를 채운 뒤 사진을 첨부하면 텍스트가 같이 전송돼 **결과가 섞이는 버그**가 흔하다 → 첨부 시 텍스트 비우기.
- 칩/버튼 라벨은 짧게(`예시: 지급명령서`→`지급명령서`) — 한 줄에 들어가게. 버튼이 줄바꿈되면 size를 medium으로.

## 3. 모바일 화면 검증법
- **폰 WebView는 `adb screencap`이 검은 화면**(보안 서피스)으로 나온다.
- → **playwright로 모바일 뷰포트(390px) 캡처**: `newPage({ viewport: { width: 390, height: 844 }, isMobile: true })`. fold(스크롤 0)·full 둘 다 본다. 파일 첨부 상태는 `setInputFiles`로 시뮬레이션해 검증 가능.
- **실기기·실입력으로 도그푸드하라**(Stage 07): 데스크톱에선 안 보이던 게 폰에서 드러난다 — 첫 화면 위치 밀림, 키보드 가림, 사진 입력 흐름, 실제 데이터의 매칭 품질.

## 4. 한 줄 요약
반응형 웹을 그대로 쓰지 말고 **모바일만 앱 셸로 재구성**(`md:` 분기) → 입력 우선, 한글 줄바꿈, 사진 입력 UX, playwright 검증. 데스크톱은 보존.
```

## 2. AI 매칭/분류 결정적 안전망 패턴
**대상: `docs/AI_Hackathon_Operating_System.md` — §1.2("좋은 AI 프로젝트…") 바로 뒤에 아래 절 삽입**

```markdown
## 1.2.1 좋은 AI 프로젝트는 '자유 생성'을 결정적 규칙으로 가둔다 (매칭 안전망)

AI의 강점(임의 입력 해독·언어→개념 변환)은 살리되, **분류·매칭·사실은 AI의 자유 생성에 맡기지 않는다.** AI 자유 분류는 흔들린다 — 실제로 "자동차세 고지서"를 받고 관청을 **'보건소'로 잘못 분류**한 사례가 있었다(엉뚱한 출처가 컨텍스트에 끼어들어 모델을 오도).

**원칙: 고빈도·고위험 결정은 결정적(deterministic) 규칙으로 보정한다.**
- **키워드 안전망**: 자유 라벨 대신 **대분류 테이블 + 키워드 매핑**으로 사안→관청·처리 채널·공식 URL을 결정적으로 주입(예: 자동차세→시군구청+위택스). 새 영역은 한 줄 추가로 확장.
- **AI 출력까지 스캔하라**: 입력이 **이미지/사진**이면 텍스트가 비어 키워드가 안 걸린다 → AI가 만든 요약·관할 텍스트까지 함께 스캔해야 잡힌다.
- **출처는 화이트리스트에서만**: 답·URL은 검증된 코퍼스에서만 인용(환각 차단). 근거 없으면 "공식 확인 필요".
- **검증된 자원만 연결**: 외부 링크(처리 채널 등)는 **실제 접속 확인** 후 넣는다(죽은 링크 주의).

이건 행정뿐 아니라 분류·검색·매칭이 들어가는 **대부분의 AI 프로젝트**에 적용되는 신뢰 원칙이다. 심사위원은 "AI가 틀릴 때 어떻게 막는가"를 본다(§2.3 의심). "똑똑함"이 아니라 **"정직함(틀려도 엉뚱한 안내가 안 나간다)"**이 차별점이 된다.
```

## 3. 위치 기능 3대 함정
**대상: `docs/kit-assets.md` — "KRDS 주의사항" 섹션 뒤(다음 `---` 앞)에 아래 섹션 삽입**

```markdown
## 위치 기능 함정 (주변 찾기 — 실측)
`NearbyOffices`/`KakaoMap` 등 위치 기능을 쓰는 모든 주제에 해당. 데스크톱·도심에선 안 보이다가 실기기·농촌·관할 사안에서 드러난다.
- **농촌 반경**: 카카오 keywordSearch 반경 5km는 **군 단위에서 0건**(가까운 군청/시청이 5km 밖). → 반경 **20km(카카오 최대) + 반경 제거 전국 최근접 폴백**.
- **관할 vs 최근접 (개념 위반 주의)**: "내 주변 가장 가까운"은 **관할이 정해진 사안엔 틀린다** — 자동차세는 차량 등록지(예: 무안군청)지 현재 위치 근처(목포시청)가 아니다. concept이 "관할은 주소·사건 기준"이라면 **관할 고정 사안은 지도를 숨기고 처리 채널(위택스 등) 버튼**으로, 어디서나 가능한 사안(전입·복지)만 최근접 지도. (사안별 `locationMode` 분기.)
- **http에서 geolocation 차단**: `navigator.geolocation`은 **보안 출처(https)가 아니면 막힌다** — OS 권한을 줘도 안 됨. 네이티브 앱(http localhost 로드)·로컬 http에선 `@capacitor/geolocation` 네이티브 플러그인으로 우회하거나 https 배포.
- **카카오 도메인 등록**: JS SDK는 페이지 **origin을 도메인에 등록**해야 동작(미등록 referer→401). server.url이 바뀌면(localhost/공인IP/배포) 그 origin을 등록. 실제 origin은 `console.log(location.origin)`로 확인.
```

## 4. 폰 앱 시연 — scrcpy 미러링
**대상: `docs/android-apk-build.md` — 문서 끝에 아래 절(§10) 추가**
> (참고: `docs/android-apk-build.md` 자체가 이 세션 산출물인 APK 빌드 레시피다. main에 그 파일이 없으면 WORK2 브랜치의 전체본을 가져온 뒤 이 절을 포함시킬 것.)

```markdown
## 10. ★ 시연: 폰 화면 미러링 (scrcpy)
빔프로젝터에 **폰 앱이 실시간으로 움직이는 걸** 그대로 보여주는 표준 방법. AI가 기기화면을 못 봐도 **사람이 발표 때 직접** 쓴다.
- **scrcpy** = 안드로이드 화면을 USB로 노트북에 저지연 미러링(+노트북 마우스로 폰 조작). 무료·오픈소스·인터넷 불필요. 이미 깔린 adb를 그대로 쓴다.

    winget install --id Genymobile.scrcpy -e   # 1회 설치(~15MB)
    # 시연 시(폰 USB 연결 상태):
    $env:ADB = 'C:\Android\sdk\platform-tools\adb.exe'   # scrcpy가 우리 adb를 쓰게(서버 충돌 방지)
    & $env:ADB reverse tcp:3000 tcp:3000                  # 앱이 PC dev 서버 접속(USB 터널)
    scrcpy                                                # 폰 화면 미러 창 열림 → 이 창을 빔으로

- **주의**: `--window-title` 등 옵션에 **한글/공백 인자**를 주면 `Start-Process`에서 쪼개져 실패한다 — 옵션 없이 띄우거나 공백 없는 값 사용.
- 전제: `web`에서 **dev 서버 실행 중** + USB 케이블 연결 + `adb reverse`. (server.url=localhost 방식이라 케이블이 끊기면 앱이 데이터를 못 받음.)
- **시연 체크리스트**: 폰 **화면 꺼짐 시간 늘리기**·잠금 해제 유지 · scrcpy 창을 빔 출력 화면에 배치 · 사진 찍어 올리기→분석→결과 카드→**상단바 내려 알림** 보여주기 · 라이브 흔들리면 캡처(`demo/`)로 전환.
- 무선(같은 와이파이)도 가능하나(`scrcpy --tcpip`) **USB가 가장 안정적** — 발표엔 USB 권장.
```
