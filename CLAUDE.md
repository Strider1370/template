# CLAUDE.md — 한국 공공 서비스 해커톤 키트

주제를 받으면 "돌아가는 데모 + 발표"까지 만드는 출발점. **진행 체크리스트 + 참고 라이브러리 + 빈 스캐폴드**로 구성.

## 쓰는 법
- **`PROCESS-CHECKLIST.md`** 를 따라 13단계를 진행한다.
- 각 단계에서 필요한 `reference/` 문서를 펼쳐 적용한다.
- 결과물(앱·발표)은 주제를 받고 새로 만든다.

## 참고 라이브러리 (`reference/`)
| 파일 | 내용 |
|---|---|
| `01-기획-인사이트.md` | 문제정의·Insight·JTBD·차별화·레드팀·발표 내러티브 |
| `02-기획-병렬리서치.md` | LLM 병렬 리서치(브리핑→발표맥락 포함→트랙 전파→통합) |
| `03-웹스캐폴드-KRDS.md` | web/ 스캐폴드·KRDS·모바일 디자인 |
| `04-발표-Slidev.md` | Slidev 발표 제작·카피 작성·톤 |
| `05-배포-모바일-APK.md` | 자체 AWS https 배포·APK·scrcpy 미러링 |
| `06-AI연동.md` | LLM 키 런타임 입력·음성(GPT Realtime) |

## 자산
- `web/` — Next.js + KRDS 빈 스캐폴드(정체성 자리표시). 폼·지도 등 부품은 필요할 때 추가.
- `data/` — 키 없이 쓰는 공개데이터(인덱스 `data/INDEX.md`).
- `presentation/` — Slidev 발표 생성 도구. `npm run presentation:build`.

## 원칙
- 구현 안 한 기능 발표 금지, 근거 없는 수치 금지. **판정·계산은 코드**, AI는 이해·대화·값 추출.
- 발표 시간 50%+ 는 데모+킥. 시간 부족 = 단계 생략❌ → 기능 범위 축소⭕.
- **LLM 키 = 앱 런타임 직접 입력**(⚙️·⋯ → 키 모달), 없으면 폴백. 코드/저장소에 키 금지.
- 모바일 반응형 필수(반응형 ≠ 앱).
- 대회: 신규 저장소·점진 커밋·출처 기록. 결과물은 당일 제작.
