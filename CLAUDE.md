# CLAUDE.md — 한국 공공 서비스 해커톤 키트

주제를 받으면 "돌아가는 데모 + 발표"까지 만드는 출발점. **진행 체크리스트 + 참고 라이브러리 + 빈 스캐폴드**로 구성.

## 쓰는 법
- **`PROCESS-CHECKLIST.md`** 를 따라 13단계를 진행한다.
- 각 단계에서 필요한 `reference/` 문서를 펼쳐 적용한다.
- 결과물(앱·발표)은 주제를 받고 새로 만든다.
- **모듈 적재**: 시작은 `0-base`만. 단계가 오면 무거운 번들을 루트에 투입한다 — **05 빌드→`2-build`**, **09 발표→`3-present`**, **12 배포→`4-deploy`**. (매핑·복원 명령: `PROCESS-CHECKLIST.md` "모듈 적재" + `modules/README.md`)

## 자산
- `web/` — Next.js + KRDS 빈 스캐폴드(정체성 자리표시). 폼·지도 등 부품은 필요할 때 추가.
- `data/` — 키 없이 쓰는 공개데이터(인덱스 `data/INDEX.md`).
- `presentation/` — Slidev 발표 생성 도구. `npm run presentation:build`.

## 원칙
- 구현 안 한 기능 발표 금지, 근거 없는 수치 금지. **판정·계산은 코드**, AI는 이해·대화·값 추출.
- 발표 시간 50%+ 는 데모+킥. 시간 부족 = 단계 생략❌ → 기능 범위 축소⭕.
- **LLM 키 = 앱 런타임 직접 입력**(⚙️·⋯ → 키 모달), 없으면 폴백. 코드/저장소에 키 금지.
- 모바일 반응형 필수(반응형 ≠ 앱).
- 대회: 신규 저장소·**산출물마다 점진 커밋 + 백그라운드 푸시**(흐름 안 막게)·출처 기록. 결과물은 당일 제작.

## AWS 작업 전 (서버·배포·APK) — 자격증명 게이트
- 첫 명령 전에 `aws sts get-caller-identity` 확인. 실패하면 **멈추고** 사용자에게 "터미널에서 직접 `aws configure`"만 요청한다.
- **사용자가 하는 건 `aws configure` 한 번뿐.** 시크릿 키·공인 IP는 AI에게 주지 않는다(AI가 `~/.aws`에서 읽고, IP는 자동 감지). 시크릿을 채팅·env·git·명령에 절대 적지 않는다. (절차: `reference/05 §3`)

## 소통
- 초보자에게 설명하듯 — 결론만 던지지 말고 무엇을·왜·트레이드오프를 풀어서. 새 제안 앞엔 배경 1~2문장.
- **단계마다 승인받고 진행.** 한 단계 끝내면 ① 짧게 요약 ② "다음 단계로 갈지" 확인. 받은 승인은 **다음 한 단계까지만** — "진행" 한마디로 여러 단계 몰아 처리 금지. 명시적 위임("끝까지 알아서")일 때만 연속 진행.
