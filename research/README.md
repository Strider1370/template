# research/ — 리서치 보고서 둘 곳

병렬 리서치(방법은 `reference/02-기획-병렬리서치.md`)로 만든 **트랙별 보고서 + 통합 보고서 + 출처 로그**를 여기 남긴다.

- 트랙별 md 보고서(예: `jtbd.md` · `domestic.md` · `overseas.md` · `feasibility.md` · `judge-review.md`) + 통합 보고서 1개.
- **`sources.json`** = 출처 모음(제출용 출처 목록의 원본). 검증 가능한 모든 주장을 출처와 연결한다 — **근거 없는 수치 금지**.
- 파일명·형식은 자유. 중요한 건 읽기 쉬운 보고서 + 주장-출처 연결.

> 이 폴더는 대회 당일 채운다(사전 제작물 아님). 공개데이터 출처는 `data/data-sources.md` 참고.
