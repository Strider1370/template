# Stage 01 · 트랙 B — 국내 사례 리서치

> 주제: **임산부·신생아 원스톱 지원** (임신주차/출산예정일/거주지 입력 → 시기별 정부 지원을 "지금 받을 것 → 곧 받을 것" 순서로 자동 정렬·안내).
> 결론 먼저: **시기순 정렬 + 거주지 자동 판단**이라는 조합은 기존에 비어 있는 틈이다. 그러나 "임신주차별 정보 제공"과 "원스톱 신청"은 이미 정부 앱(아이마중)·민간 앱(마미톡)이 상당 부분 구현해 둠 → 차별점을 *정보 제공*이 아니라 *시기순 자격 정렬·우선순위*에 둬야 중복을 피한다.

---

## 1. 이미 존재하는 기능 (무엇을 제공 / 무엇을 못 하는가)

### 1.1 정부24 — '맘편한 임신' 원스톱 서비스 (공공)
- **제공**: 임신 단계 지원을 한 화면에서 통합 신청. 엽산제·철분제, 표준모자보건수첩, KTX/SRT 임산부 할인, 임신·출산 진료비(국민행복카드), 청소년 산모 의료비, 산모·신생아 건강관리, 에너지바우처, 난임부부 시술비 등. [src-01][src-08]
- **소득/대상 차등 서비스 존재**: '임신출산 진료비(의료급여수급자)', '에너지바우처(생계·의료급여 수급자)', '청소년 산모(만 19세 이하)' 등은 **자격 요건이 다름**. [src-08]
- **못 하는 것**: 페이지·신청 흐름에 **자격 자동판단/맞춤 안내가 명시돼 있지 않다**(unverified-as-absent — 페이지에서 해당 기능 언급 없음). 사용자가 "내가 무엇에 해당하는지" 스스로 판단해야 함. **임신 주차/예정일 기준 시기순 정렬도 없음** — 받을 수 있는 목록을 한 번에 펼쳐 보여줄 뿐. [src-08]

### 1.2 정부24 — '행복출산' 원스톱 서비스 (공공)
- **제공**: 출생신고와 함께 출산 지원을 통합 신청. 전국공통(첫만남이용권·부모급여·아동수당·양육수당·출산가구 전기료 경감·다자녀 공공요금/KTX·SRT 할인) + **지자체 서비스(출산지원금·산후조리비·출산용품 등)**. [src-02][src-03]
- **못 하는 것**: 출생 *이후* 신청 시점에 묶여 있음(임신기~출산 직후를 잇는 연속 안내 아님). 지자체 서비스는 "연계/자격확인이 안 되면 별도 서류 필요"라는 단서가 붙고, 지자체별 실시 여부·금액·신청기한이 제각각이라 **사용자가 거주지 기준으로 따로 확인**해야 함. [src-02][src-06]

### 1.3 아이마중 앱 (보건복지부/한국사회보장정보원, 공공) — **우리 컨셉과 가장 근접**
- **제공**: 임산부 등록 + 온라인 민원(임산부 지원 5종·의료비 지원 6종) + **임신 주차별(3~40주)·아이 월령별(1~36개월) 맞춤 건강정보** + 정부지원정책 안내 + 임산부/아기수첩. 2023년 3월 출시. [src-04][src-05]
- **못 하는 것(공식 소개 페이지 기준 명시 없음 = 부재로 해석)**: **자격 자동판단 없음**, **지자체별 맞춤 안내 없음**, **"지금 받을 것/곧 받을 것" 시기순 우선순위 정렬 없음**. 주차별 *건강·생활 정보*는 시기별이지만, *받을 수 있는 제도·지원금*을 시기순으로 줄 세워 주지는 않음. [src-05]
- → **중요**: "임신 주차별 + 정부지원 통합"이라는 큰 틀은 이미 정부 앱이 점유. 우리가 단순히 "주차별 정보 + 지원 목록"을 만들면 **아이마중과 중복**된다.

### 1.4 복지로 (보건복지부/한국사회보장정보원, 공공)
- **제공**: 복지서비스 찾기 + 온라인 신청 + **맞춤형급여안내(복지멤버십)** + **복지서비스 모의계산(자가진단)**. 연령·가구구성·경제상황 기준 맞춤 안내, 소득·재산 입력 시 수혜 대상 여부 확인. [src-07]
- **못 하는 것**: 복지 전반(전 생애)을 다루는 범용 포털 → **임신·출산·영아기에 특화된 시기순 흐름이 아님**. 모의계산은 입력 부담이 크고(가구원·소득·재산·부양의무자), 결과는 "참고용". 임신 주차/예정일 같은 **시간축 개념이 없다**. [src-07]
- → 시사점: "자격 모의판단"은 복지로가 이미 가짐. 우리 차별점은 *판단 그 자체*보다 *시기축 위에 정렬*하는 것.

### 1.5 임신육아종합포털 '아이사랑' (교육부/한국사회보장정보원, 공공)
- **제공**: 어린이집 검색·입소대기 신청·보육료 결제(아이사랑카드)·시간제보육 신청 + 임신/출산/육아 상담 예약. [src-09]
- **못 하는 것**: 무게중심이 **보육시설/어린이집**에 있음. 임신기 정부지원 시기순 안내는 핵심 기능이 아님. [src-09]

### 1.6 보건복지상담센터 129 (보건복지부, 공공)
- **제공**: 전화(129, 3번→출산·아동), 카카오톡 채팅, 챗봇, 수어상담. 평일 9~18시, 일부 24시간. [src-10]
- **못 하는 것**: **사람이 응대하는 상담** — 개인화된 자동 정렬·자격판단 결과물을 즉시 주는 도구가 아님. 사용자가 질문을 해야 답이 나옴(pull 방식). [src-10]

### 1.7 민간 앱 — 마미톡 / 맘큐 등
- **제공(마미톡)**: 초음파 영상 공유, 육아·성장 기록(수유·수면·배변), 전문의 시기별 검사·정보, 클래스, 커뮤니티, 쇼핑몰. [src-11]
- **못 하는 것**: **정부지원 신청·자격판단은 핵심이 아님**(콘텐츠·기록·커뮤니티·커머스 중심, 일부 유료화). 시기별 *건강정보*는 강하지만 *제도/지원금 시기순 정렬*은 제공하지 않음. [src-11]

## 2. 공공 vs 민간 차이 요약
| 축 | 공공(정부24·아이마중·복지로) | 민간(마미톡 등) |
|---|---|---|
| 강점 | 실제 신청·통합처리, 정확한 제도 정보 | UX·콘텐츠·기록·커뮤니티 |
| 약점 | 시기순 정렬·우선순위 부재, 거주지 차등 안내 약함, 입력 부담 | 정부지원 자격판단·신청 안 다룸 |
| 자격 자동판단 | 복지로 모의계산만(범용·시간축 없음) | 없음 |
| 시간축(임신주차/예정일) 기반 정렬 | **없음**(아이마중은 *건강정보*만 주차별) | 건강정보만 주차별 |

## 3. 사용자가 실제로 겪는 마찰 (검증된 근거 기반)
1. **제도 산재**: 임신기·출산·영아기 지원이 정부24/복지로/아이마중/지자체 페이지로 흩어져 있어 한 곳에서 "내 시점에 뭘 받지?"를 못 봄. [src-01][src-02][src-07]
2. **시기별 안내·정렬 부재**: 어느 서비스도 "지금 신청해야 할 것 → 곧 신청할 것"을 **마감·신청기간 기준 시간순**으로 정렬해 주지 않음. 예: 산모·신생아 건강관리는 "출산예정일 40일 전~출산 후 30일"처럼 **신청 창(window)**이 있는데, 이를 놓치기 쉬움. [src-12]
3. **자격 자동판단 약함**: 소득·연령·청소년 산모 등 조건별 차등이 있으나 통합 신청 화면이 "당신은 X에 해당"이라고 자동 판정해 주지 않음(복지로 모의계산은 별도·범용·입력 과중). [src-07][src-08]
4. **지자체별 상이**: 출산지원금·산후조리비 등은 지자체마다 실시 여부·금액·**거주기간 단서(예: 부모 중 1인 6개월 이상 주민등록)**·신청기한이 달라 사용자가 직접 거주지 페이지를 뒤져야 함. [src-06]

## 4. 국내 규제·데이터 제약
- **개인정보/본인인증**: 실제 신청은 간편인증·공동/금융인증서 본인인증 필요 → 해커톤 데모에서 *실제 신청 연동*은 무리. **자격판단·안내(정보 제공)까지만** 데모 범위로 잡는 게 현실적. [src-01][src-02]
- **데이터 출처**: 전국공통 제도(부모급여·첫만남이용권 등)는 비교적 표준화돼 있으나, **지자체 출산지원금은 단일 공식 API가 약하고 페이지별 산재** → 데모는 소수 지역 샘플 데이터로 시연 권장. [src-06]
- (unverified) 지자체 출산지원금 통합 공공데이터셋의 최신성·완결성은 확인되지 않음 — 데모 시 "샘플/예시" 표기 필요.

## 5. 해커톤에서 중복될 가능성이 높은 접근 (피해야 할 것)
- ❌ "임신 주차별 정보 + 정부지원 목록을 보여주는 앱" → **아이마중이 이미 함**. 단순 재현이 됨. [src-05]
- ❌ "정부지원 안내 챗봇/Q&A" → 129 챗봇·복지로가 이미 점유, pull 방식 차별성 약함. [src-07][src-10]
- ❌ "복지 자격 모의계산기" → 복지로 모의계산과 중복. [src-07]
- ❌ "출산지원금 지역별 비교 표" → 블로그/민간 정리글이 이미 다수 존재(차별성 낮음). [src-06]

---

## 6. 비어 있는 틈 (Gap) — 우리 서비스의 자리
기존 서비스는 받을 수 있는 제도를 **목록(pull)**으로 펼쳐 주지만, **"내 임신주차·예정일·거주지 기준으로 *지금 마감 임박한 것 → 곧 열릴 것*을 시간순으로 줄 세워 push"** 해 주는 도구는 없다. 즉 **(a) 시기축 우선순위 정렬 + (b) 신청 창(window)·마감 리마인드 + (c) 거주지 기반 지자체 자동 매칭** 세 가지의 *결합*이 빈 틈이다. 아이마중은 (시기별 건강정보)만, 복지로는 (범용 자격판단)만, 정부24는 (통합 신청)만 가지며 — 이 셋을 **시간축 위에서 한 줄로 엮은 제품은 없다.**
