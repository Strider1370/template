# JTBD 탐색 — "우리 가족 맞춤 정부 혜택 찾기"

## 1. 한 줄 요약
사용자는 "내가 받을 수 있는 혜택이 무엇인지 빠르고 확실하게 알고, 놓치지 않고 싶다"는 일을 고용한다 — 한국 복지는 본인이 신청해야 받는 **신청주의** 구조라, 제도가 많아도(복지멤버십 안내 대상만 128종) 무엇을 받을 수 있는지 몰라서 못 받는 **사각지대**가 핵심 통증이다.

## 2. 페르소나별 JTBD

### 페르소나 A — 신혼·영유아 가구 (예: 30대 초보 부모)
| 필드 | 내용 |
|---|---|
| **Functional Job** | 출산·육아와 관련해 우리 가구가 받을 수 있는 현금성/서비스성 혜택(부모급여, 아동수당, 첫만남이용권, 지자체 출산지원금 등)을 한 번에 확인하고 신청 방법을 알고 싶다. |
| **Emotional Job** | "아이에게 줄 수 있는 걸 빠뜨리지 않았다"는 안심. 정보 못 찾아 손해 봤다는 불안 해소. |
| **Social Job** | 똑똑하고 책임감 있는 부모로 보이고 싶다. 다른 부모와 정보 공유 시 "이미 다 챙겼다"고 말할 수 있길 원함. |
| **Current Alternative** | 맘카페·지인, 토스/뱅크샐러드 알림, 복지로 직접 검색, 주민센터 출산 안내. |
| **Failure Cost** | 지자체별로 다른 출산지원금/육아용품 지원을 몰라 놓침. 신청 기한 경과로 영구 수급 불가. 카페 정보는 지역·시점이 안 맞아 오판. |
| **Success Criteria** | 우리 지역·소득 기준으로 받을 항목 목록과 "어디서 어떻게 신청"이 한 화면에 나오고, 빠진 게 없다는 확신. |

### 페르소나 B — 청년 1인 가구 (예: 사회초년생·구직 청년)
| 필드 | 내용 |
|---|---|
| **Functional Job** | 주거(월세 지원)·취업·소득 관련 청년 대상 지원을 내 자격으로 받을 수 있는지 확인하고 싶다. |
| **Emotional Job** | "나만 못 챙기고 있나" 하는 불안 해소, 자립에 대한 통제감. |
| **Social Job** | 부모/주변에 의존하지 않고 스스로 제도를 활용하는 독립적인 성인으로 보이고 싶다. |
| **Current Alternative** | 검색 포털, 정부24/복지로 직접 탐색, 친구 추천, SNS 정보. |
| **Failure Cost** | 청년 지원은 주거·고용에 분산되어 있고 자격요건(연령·소득·거주기간)이 제각각 → 정보 부족과 "기준 엄격해 안 될 것 같다"는 인식으로 아예 미신청. 청년 1인가구는 사회경제적 상황이 매우 이질적이어서 일괄 안내가 안 맞는 경우가 많음. |
| **Success Criteria** | 복잡한 자격요건을 입력 몇 개로 걸러주어, "신청해볼 가치가 있는 것"만 추려지는 것. |

### 페르소나 C — 노년·기초연금 대상 (또는 대신 알아봐 주는 자녀)
| 필드 | 내용 |
|---|---|
| **Functional Job** | 부모(또는 본인)가 받을 수 있는 기초연금·노인일자리·의료/돌봄 서비스를 확인하고 신청 절차를 알고 싶다. |
| **Emotional Job** | 부모를 잘 챙기고 있다는 안도감, 죄책감 해소. 노년 당사자는 "폐 끼치지 않고 받을 건 받는다"는 존엄감. |
| **Social Job** | (자녀) 효도하는 자녀로 보이고 싶다. (당사자) 자식에게 기대지 않는 떳떳함. |
| **Current Alternative** | 주민센터 방문 문의, 자녀가 대신 검색, 동네 어르신 입소문. |
| **Failure Cost** | 온라인 접근성이 낮아 신청주의 구조에서 가장 쉽게 누락. 정보가 흩어져 있어 자녀도 전체 그림을 못 봄. 디지털 안내책자(중장년·노령층 가이드)가 있어도 어디부터 봐야 할지 모름. |
| **Success Criteria** | 어려운 용어 없이, 받을 항목과 신청 창구(주민센터/온라인)가 명확히 안내되어 대신 신청까지 매끄럽게 이어짐. |

### 페르소나 D (선택) — 실직·저소득 위기 가구
| 필드 | 내용 |
|---|---|
| **Functional Job** | 갑작스러운 소득 상실 상황에서 긴급복지지원·생계급여 등 즉시 받을 수 있는 안전망을 빠르게 파악. |
| **Emotional Job** | 위기 속 "기댈 곳이 있다"는 안도, 막막함 해소. |
| **Social Job** | 가족을 지키는 가장으로서의 책임 수행. 도움받는 것이 부끄럽지 않다는 정당화. |
| **Current Alternative** | 주민센터, 129(보건복지상담), 지인. |
| **Failure Cost** | "기준이 엄격해 신청해도 안 될 것"이라는 인식(미신청 사유 중 가장 높게 보고된 19.3% — 단, 출처 정밀도 낮음, *unverified*)과 정보 부족으로 위기 시점에 못 받음. 사각지대 비극 사례(수원·익산 모녀 등)의 전형. |
| **Success Criteria** | 위기 상황 입력 즉시 "지금 받을 수 있는 것"과 연락처가 뜨고, 자격 가능성을 솔직하게 보여줌. |

## 3. 공통 인사이트 (가장 강한 미충족 니즈)

1. **"몰라서 못 받는다"가 1차 통증 — 정보 통합·발견 니즈.** 제도는 많지만(복지멤버십 안내만 128종, 중앙+지자체+민간 분산) 사용자는 자기 가구에 맞는 전체 목록을 한눈에 못 본다. → 입력 기반 **맞춤 필터 + "숨은 혜택 발견"** 가치가 가장 강함.

2. **자격 오판으로 인한 자기검열 — "신청해도 안 될 것 같아서" 미신청.** 기준이 엄격해 보여 시도조차 안 한다. → 정확히 거르되 **"신청해볼 가치 있음"을 긍정적으로 제시**해 자기검열을 줄이는 UX가 중요.

3. **찾기에서 끝나지 않는 행동 전환 니즈 — "어디서·어떻게 신청".** 목록만으로는 신청주의 장벽을 못 넘는다. → 혜택 목록 옆에 **신청 창구/방법 안내**를 붙이는 것이 데모의 Wow(intake initialAngle B/C와 정합).

> 데모 함의: 입력(가구·지역·소득) → 맞춤 혜택 목록 + "놓치고 있던 N건" + 신청 방법, 이 세 가지가 JTBD를 직접 때린다.
