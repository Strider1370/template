## 0. 한 문장 결론

**"흩어진 행정정보를 시민 언어로 묶어, 출처를 보여주며 '그래서 다음에 뭘 하면 되는지'까지 끝까지 안내하는 신뢰 가능한 레이어."**
→ 구현은 **C(쉬운 말)를 확실한 바닥으로 깔고, B(다음 할 일 단계화)를 Wow로 얹고, 둘 다에 '출처 인용(신뢰)'을 박는** 단일 시나리오. A는 곁가지.

5개 트랙이 독립적으로 조사했는데 **같은 결론에 수렴**했다 — 이건 강한 신호다.

## 1. 사용자 문제 (트랙 A) — 누가, 왜 막히나

- **메인 세그먼트: 첫 민원 처리하는 청년**(이사·전입·취업·자취). first-time이라 절차 자체가 낯섦.
- 보조: 복지 신청 고령자/보호자, 한국 행정이 낯선 외국인 주민, 소상공인 창업자.
- **핵심 JTBD(메인):** "처음 하는 행정 절차를 실수 없이 끝내고 싶다" — 기능적으로는 *어디서·무슨 서류로·어떤 순서로*, 감정적으로는 *불안을 확신으로*.
- **사용자가 직접 말한 3대 문제와 검증:**
  - ① 처음이라 낯섦 → **타당**(모든 세그먼트 Job의 출발점).
  - ② 법·용어 낯섦 → **강하게 타당**. 국립국어원 '필수 개선 행정용어 100개' 등 정부도 공식 문제로 인정(어려운 행정용어 시간비용 연 약 118.3억 원 보고 — *발표 인용 전 원문 재확인 필요*).
  - ③ 정보 과잉·신뢰 판단 불가 → **타당(정성)**. 정량 수치는 권위 출처로 확정 못 함 → **수치로 주장하지 말고 "공식 출처를 함께 보여준다"는 기능으로** 표현.

## 2. 국내 현황 (트랙 B) — "이미 다 있다", 빈틈은 어디

- A·B·C 세 방향 모두 **이미 공공 서비스로 존재하거나 출시 임박**: 정부24, 복지로 복지멤버십(163종), 국민비서 구삐, 보조금24, 120다산콜/서울톡, 공공언어 정비. 특히 **'AI 정부24'가 B를 거의 똑같이 준비 중(2026 1분기 시범·4분기 정식)** — 지금은 공백.
- **민간 성공 공식(토스·카카오):** "공공 데이터 + 시민 언어 UX". 데이터 독점이 아니라 **경험 레이어**로 승부 → 우리도 같은 길.
- **가장 큰 빈틈(우리가 노릴 곳):** ④ 절차 불투명 + ① 파편화 횡단 + ⑦ 신뢰. 기존 서비스는 **'조회까지'만 하고 신청은 외부 점프**, 행정 언어 그대로, 출처·최신성 표시 없음.
- **중복되면 지는 접근(피하라):** 단순 Q&A 챗봇, 지원금 찾기, 알림받기, 단발 공문 요약.

## 4. 구현 현실성 (트랙 D) — 4시간에 되는가

- **된다.** 단일 페이지 웹앱 + 키트의 `/api/llm` RAG. C가 데이터 의존 최소(LLM 단독 동작) = **확실한 바닥**, B를 Wow로 얹기.
- **데이터 현실(중요):** 민원 절차(구비서류·처리기간)의 **정형 단일 API는 사실상 없다**(민원처리기준표는 관보/HTML 고시). → B는 **권위 출처 기반 큐레이션 fixture 20~40건**이 정답(품질 저하 아님, 도메인 성격). 검색 모집단은 보조금24 스냅샷(~10,957) 전수.
- **실시간 API 불필요**(정적 데이터). 외부 의존은 **OPENAI_API_KEY 한 곳뿐** → 키 없으면 C=규칙 치환, B=fixture 카드 직접 렌더하는 **폴백이 핵심**.
- **자산 재사용:** `web/` 스캐폴드 · `web/lib/llm.ts`(두뇌) · KRDS+엠블럼(신뢰 룩) · `regions.ts`(관할 필터) · 배너 생성기. (지도·대피소는 안 씀.)

## 5. 심사위원·차별화 (트랙 E)

- **최대 리스크:** "또 GPT 붙인 행정 챗봇" 30초 격하. **의심 ①(정부24와 뭐가 다른가)·②(환각)**를 데모 첫 화면에서 **선제 방어**.
- **차별화 3중 결합(하나라도 빠지면 기존 서비스로 대체됨):**
  **① 신뢰(출처 인용·모름 인정) ★ + ② 다음 할 일 단계화 ★ + ③ 쉬운 말.**
- **포지셔닝 한 문장:** *"정부24는 '신청 버튼'을 줍니다. 우리는 그 버튼이 어디 있는지조차 모르는 사람을 거기까지 데려다줍니다."*

## 6. 킥(kick) 후보 종합 — Stage 02 입력 (확정 아님)

> Gate 요구: 킥 후보 ≥1. 아래는 수렴된 추천 묶음.

- **★ 최강 데모 킥 (5개 트랙 공통 추천): C-1 + B-1 + 출처 배지를 한 화면에**
  - **C-1:** 어려운 공문 붙여넣기 → 좌(원문)/우(쉬운 말) 대조 + "그래서 당신이 할 일 1·2·3".
  - **B-1:** 막연한 일상어 한 줄("이사했는데 전입신고 뭐부터?") → [어디서·무엇을 들고·순서·다음 할 일] 카드.
  - **신뢰 레이어(K6/K7):** 답변마다 **출처 칩**, 근거 없으면 **"확인 필요 + 공식 링크"**로 폴백(NYC 실패 회피).
- 보조 킥(시간 남으면): K4 관할 라우팅(regions.ts), K5 "한 입력 → 연관 절차 펼침"(Wow 큼).
- A(자격 사전판정 위저드)는 복지멤버십과 중복 + 개인정보 리스크 → **익명 추정 수준 곁가지만.**

## 7. Stage 02로 넘기는 결정 질문 (사용자 승인 사항)

1. **세그먼트:** '첫 민원 청년' 1개 집중 vs 멀티세그먼트 확장 시연.
2. **B/C 비중:** 권고 = "C 바닥 + B Wow + 신뢰 박기" 단일 시나리오. 동의/수정?
3. **A 포함도:** 보조(익명 추정)만 vs 빼기.
4. **데모 시나리오 민원 1개:** 전입신고? 청년월세? 출산 후속절차? (큐레이션 fixture 대상 확정용)
5. **신뢰 표현:** 수치 주장 금지 → "출처 인용 + 기준일 표시"를 핵심 가치로. 동의?

## 8. 위험·주의 (다음 단계가 알아야 할 것)

- 'AI 정부24'(2026) 정면 중복 위험 → **횡단·액션·신뢰의 각**으로만 차별화 유지.
- LLM 환각 = 이 도메인의 치명상 → **RAG 출처 강제 + '모름' 폴백** 없으면 전 차별화 붕괴.
- 행정용어 시간비용(118.3억) 등 **검색 요약 수치는 발표 전 원문 재확인**. 근거 없는 수치·모집단 축소 거짓 금지.
- 데이터는 전수 적재(모집단)·표시 일부 원칙. 시간 부족 시 데이터가 아니라 화면 수를 줄인다.
