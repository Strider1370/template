# Stage 11 — Rehearsal (리허설)

## 1. 목적
실제 낭독 시간을 측정하고, 라이브 데모 ↔ 스크린샷 백업 전환을 점검하며, Q&A를 마지막으로 다듬는다. **사용자 최종 승인**으로 발표를 확정한다.

## 2. 시작 조건
- Stage 10 Gate 통과, 검증된 발표자료 + 데모 스크린샷(백업) 존재.

## 3. 이번 단계에서 반드시 읽을 파일
- `presentation/slidev/slides.md`, `presentation/script.md`, `presentation/qna.md`, Stage 07 데모 스크린샷
- guidance: `docs/AI_Hackathon_Operating_System.md` §8 "Stage 11 Review"; `docs/CLAUDE_Notion_Slidev_Integration_Guide.md` §3 "Stage 11 — 리허설 반영". 이 섹션만.

## 4. 이번 단계에서 읽지 않아도 되는 파일
- 리서치 원문, 생성/검증 상세 규칙, 무관한 구현 세부.

## 5. 필수 입력
- 발표자료 + 스크립트 + 데모 스크린샷. 없으면 시작하지 않는다.

## 6. 메인 에이전트의 역할
- 스크립트 실제 낭독 시간 측정(발표 시간 내인지). 초과면 슬라이드/문장 조정.
- 라이브 데모 시연 경로와 스크린샷 백업 전환 시점을 점검.
- Q&A 최종 보강.
- **사용자에게 최종 발표를 제시하고 승인을 받는다 — 초보자에게 설명하듯** 무엇이 바뀌었는지·발표 시간/라이브 데모/Q&A가 각각 어떤 상태인지 쉬운 말로 풀어서 제시한다(결론만 던지지 말 것). 승인 후에만 확정.

## 7. 병렬 서브에이전트 구성
- 없음.

## 8. 각 서브에이전트의 작업 계약
- 해당 없음.

## 9. 생성해야 하는 산출물
- 리허설 결과(시간·전환 점검) 기록, 갱신된 `presentation/script.md`/`qna.md`(필요 시), 사용자 승인 기록.

## 10. 파일 소유권
- 메인 전용: 발표 관련 파일 갱신 + 승인 기록.

## 11. 제한 시간
- 10분. 초과 시 1회 완주 리허설 + 시간 측정만이라도 완료(생략 X).

## 12. 완료 조건
- 낭독 시간이 발표 시간 내, 라이브↔스크린샷 백업 전환 확인, **사용자 최종 승인 완료**.

## 13. 기계적 Gate (Mechanical)
- 명령: `npm run gate:rehearsal`
- 분류: **checklist**. 리허설 기록 + 스크린샷 백업 존재 + 승인 기록 체크리스트 출력 → 자가점검.

## 14. LLM Review Gate
- `npm run cross-review -- presentation/script.md`(최종) (Codex 우선 → 클로드 폴백).
- 검토: 시간 적정, 구현↔발표 일치, 데모 실패 대비(스크린샷 백업) 완비, Q&A 충분.

## 15. 사용자 승인 여부
- **`humanApproval: true`.** 승인 전까지 멈춘다. 승인 전 Stage 12 패키징 금지.

## 16. 실패 시 폴백
- 시간 초과 시 슬라이드 삭제/병합(데모+킥 보존). 라이브 불안정하면 발표를 스크린샷 백업 중심으로 전환.

## 17. 다음 단계에 전달할 정보
- 확정 발표자료 + 데모 스크린샷 + 승인 기록 (Stage 12 패키징 입력).

## 18. 금지 사항
- 승인 없이 패키징 진행 금지.
- 시간 초과 상태로 확정 금지.

## 19. 단계 완료 보고 형식
- `workflow/templates/stage-report.md` → `workflow/history/stage-11-rehearsal.md`. "사용자 결정"에 최종 승인 내용 명시.
