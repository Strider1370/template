# LLM API 연동 방식 — "앱에서 키 직접 입력" (사용자 결정 2026-06-21)

> 템플릿 표준: 모든 AI/LLM(텍스트·음성 Realtime) 기능은 **앱 런타임에 사용자가 직접 넣은 키**로 작동한다.
> 코드·`.env`·저장소·APK에 **키를 절대 박지 않는다.** AI를 쓰는 모든 단계는 이 방식을 기본으로 유도한다.

## 왜
- 데모는 보통 **운영자(우리)가 직접 시연** → 그 자리에서 키만 넣으면 됨. 서버 토큰 통로(ephemeral) 안 만들어도 됨 = 단순.
- 키를 저장소/배포물에 안 박으니 **유출 위험 없음**(그 자리에서만 메모리에 존재).

## UI — 키 입력 위치
- 헤더(또는 우상단)에 **설정(⚙️) 또는 더보기(⋯) 아이콘** 1개.
- 누르면 **키 입력 모달/드로어** 열림: "OpenAI API 키" 입력칸 + 저장 버튼 + (입력됨/없음) 상태 표시.
- 입력하면 그 세션에서 **AI 기능 활성화**. 비우면 폴백 모드.

## 저장·보안 규칙
- 키는 **메모리 또는 `sessionStorage`** 에만. 새로고침/종료 시 사라짐(localStorage 영구저장은 지양, 쓰면 "이 기기에만" 경고).
- **커밋/배포물(.env·APK·git)에 키 금지.** 런타임 입력만.
- 데모 종료 후 그 키 **폐기(rotate)** 권장.
- ⚠️ 키 입력칸을 누구나 보므로: **운영자 시연 전제**. 불특정 다수가 자기 키를 넣는 BYOK도 같은 UI로 가능(각자 책임).

## 동작 (기존 헬퍼에 키만 주입)
- 텍스트 LLM: 기존 `web/lib/llm.ts`(`openaiChat`) + `/api/llm` 라우트를 그대로 쓰되, **런타임 키를 요청에 실어 보냄**(예: 헤더 `x-llm-key`) → 라우트가 env 대신 그 키 사용. 키 없으면 `null`→**폴백**(하드코딩 응답).
- 음성 Realtime: 클라이언트가 그 키로 직접 연결(데모용 A안). 상세 `docs/voice-realtime-notes.md`.
- **키 없을 때도 앱은 폴백으로 완전히 동작**해야 한다(WORK2 패턴: 드롭다운 입력·사전작성 설명).

## 적용 지점 (AI 쓰는 단계는 이걸 기본으로)
- **Stage 04 plan**: 키는 "사용자 사전준비(.env)"가 아니라 **런타임 입력(설정 아이콘)** 으로 계획.
- **Stage 05 build**: 설정/더보기 아이콘 + 키 모달 + 메모리 저장 + 헬퍼에 키 주입 + 무키 폴백 구현.
- **Stage 07 demo**: 시연 시작 시 키 입력 → AI 켜짐(스크립트에 한 줄).
