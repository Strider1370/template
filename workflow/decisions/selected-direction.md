# Stage 02 — 확정 방향 (사용자 승인 완료)

> 승인: 사용자 — "승인, 이대로 확정" (2026-06-19). 간판 문장: "정부는 vs 우리는".
> 북극성: [concept.md](../../concept.md). 후보 전체: [insight-candidates.md](insight-candidates.md).

## 확정 방향: "무엇을이 아니라 왜 당신이"
- **척추 Insight:** ③ 목록이 아니라 "이유"(신뢰는 근거에서) + ⑤ AI는 "판정"이 아니라 "통역사".
- **보조:** ② 무로그인·무저장(차별화 한 문장의 동등한 절반 + 4시간 데모 현실성 + 개인정보 안전), ④ 신청 동선.

## 근거 (왜 이 방향인가)
- **사용자 통증 직격:** 미수급의 핵심은 정보 부족이 아니라 "내가 자격이 될까"의 확신 부족 → "왜 당신이 해당되는지"가 그 확신을 만든다.
- **차별화:** 기존(복지로·보조금24)은 로그인·정보연계 후 '목록'만 제공. 우리는 로그인 0회 + '이유'까지.
- **AI Leverage 정당화:** AI가 (1) 자연어 입력 구조화 (2) 확정 텍스트 다듬기 두 곳에 본질적으로 쓰이되, 자격 판정·사실 생성은 금지 → "AI 없다"·"환각" 동시 방어.

## 데모 윤곽
1. 자연어 입력(로그인 없음) → 2. AI 구조화 → 3. 규칙엔진 결정론적 매칭(예시 8~10건) → 4. 카드별 ✓충족 체크리스트 + "왜?" 배지 + AI 다듬은 설명 + 신청 동선 → 5. "예시 데이터"·면책 상시.

## 레드팀 4개 조건 (구현 시 반드시 반영 → Stage 03 spec으로 전달)
1. 설명 생성 = rewrite만(사실 생성 금지) + fixture 외 값 출력 시 폴백 + 시연 입력 사전 캐시.
2. "왜"를 텍스트가 아닌 시각적 체크리스트·배지로(0.5초 인지). 가능하면 복지로 대비 컷.
3. 차별화 진술 = ③+② 묶음("로그인 0회로 '왜 당신인지'까지").
4. "가족 맞춤"→"예시 가족 시나리오", "예시 데이터" 라벨 화면 상시.

## 안 하는 것 (범위 밖)
실제 본인인증·정보연계, 전국 전수 DB, 자격 법적 확정, AI 자격 판정/사실 생성.
