# Stage 02 — 확정 방향 (사용자 승인 2026-06-18)

> `concept.md`의 근거·데모 윤곽·확정 제약. Stage 03 spec의 토대.

## 확정 방향
**방향 A — "임신·출산 지원의 시계(時計)"**
- Insight 스택: **I1 타이밍**(척추) + **I6 손실 회피**(후크) + **I4 판단 대행**(AI 필수 근거) + **I2 지역**(보조).
- 포지셔닝: 정보 카탈로그/통합신청이 아니라 **시기·지역 맞춤 행동 타임라인 내비게이션**.

## 근거 (research 연결)
- 문제: 정보 산재(`problem-01`) → 시기 마감 절벽(`problem-02`: 부모급여·첫만남 출생 60일 소급, 산모·신생아 40일전~30일후) → 자격·지역 판단난(`problem-03`).
- 빈 틈(`gap-01`): 통합신청(정부24)·주차정보(아이마중)·모의계산(복지로)은 있으나 **시기순 정렬+마감 리마인드+지자체 매칭의 결합은 없음**.
- 해외 메커니즘(`overseas-01`): UK Sure Start 윈도우 / 호주 사전신청 시간축 / 에스토니아 능동성.

## 데모 윤곽 (happy path)
1. 입력: 임신 주차(또는 출산예정일) + 거주지(시도→시군구, `regions.ts` 재사용).
2. 출력: "지금 신청 가능 → 곧 열림" **타임라인 카드** + 각 카드 **D-day** + 마감 임박 **손실 경고 점등**.
3. 보조 동작(같은 화면): 거주지 토글 → 지자체 지원금 즉시 변동(I2).
4. AI 1장면(필수): 교차 케이스 자연어 질문 → 고정 데이터 기반 자격 추림(I4).
- 키(카카오/Claude) 모두 없어도 핵심 경로는 규칙·고정데이터로 동작.

## 레드팀 보강 반영 (승인 전 적대적 검증 결과 → 확정 제약)
- **P1 (치명·필수)**: 핵심 숫자("60일 소급", 부모급여·아동수당 금액)는 현재 2차출처 → **Stage 03 spec 확정 전 정부24/복지로 1차출처로 교체**. 불가 시 발표 화법을 단정형에서 완화 + 데이터에 `asof`/`verified` 배지. 미검증 숫자 단언 금지.
- **P2 (필수)**: **AI 라이브 1장면(교차 케이스)을 데모 cut 금지 항목으로 승격.** 시간 부족 시 제도 수를 줄이되 AI 시연 1개는 남긴다. (데이터 0개여도 도는 하드코딩 시나리오 1개 확보)
- **P3 (확정)**: 제도는 **절벽 제도 3~4개**(첫만남이용권 · 부모급여 · 산모·신생아 건강관리 · 고위험임산부 의료비)로 선제 축소. 8~12개 전수 금지.
- **데모 연출 필수**: Wow는 정적 화면이면 탈락 → **입력 라이브 변경 시 타임라인 재정렬 + D-day 갱신 + 손실경고 점등**이 보여야 함(Stage 05 구현 + Stage 07 검증 대상).

## 확정 산출물
- `concept.md` (북극성, 6헤딩) — 한 문장 피치 / 페르소나(박지은 32세) / 킥(시간축 내비) / Wow(라이브 재정렬+손실경고) / 마지막 문장 / 안 하는 것.
- 마지막 문장(사용자 확정): **"받을 수 있는 지원이 아니라, 지금 받아야 하는 지원을 알려줍니다."**

실제 신청·본인인증·결제 / 전국 전 지자체 망라 / 제도 전수. (concept.md "안 하는 것"과 일치)
