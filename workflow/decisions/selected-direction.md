# Stage 02 — 확정 방향 (사용자 승인 완료)

> 승인: 사용자 "넓게 그대로"(2026-06-20). 방향 A 확정. 근거·데모 윤곽은 아래. 북극성은 `concept.md`.

## 확정 방향
**방향 A — "겁먹지 않게, 첫 걸음을 떼주는 행정 온램프."**
받은 행정 문서/막막한 상황 → (사진·붙여넣기·상황설명) → 쉬운 말 + 기한 + 첫 걸음 + 관할 + 출처 → 정부24로 핸드오프. 모바일 PWA.

## 선택 근거
- 정부24·AI정부24(2026)가 "신청(미는)" 방향을 이미/곧 점유 → 우리는 "받은 문서(받는)" 진입점으로 비켜 차별.
- 사용자 본인 경험에서 검증된 감정: "신고할 일은 드문데 막상 닥치면 겁부터 난다" = 핵심 인사이트(정보 부족이 아니라 첫 걸음 공포).
- 리서치 5트랙 + C 보강조사가 "받은 무서운 문서"(지급명령 등)의 실제 피해를 뒷받침. 해외(GOV.UK·Albert·NYC)가 메커니즘·안전장치를 증명.

## 데모 윤곽 (우선순위 빌드 — 넓은 범위, 안전장치 포함)
1. **[핵심] 지급명령 시나리오** 끝까지: 붙여넣기/타이핑 입력 → 4칸 카드(뜻·기한2주·첫걸음·관할텍스트) + 출처 배지 + 환각 폴백("근거 없으면 공식 확인 필요").
2. **[핵심] 즉석 입력 1개**: 미리 안 짜둔 문서/상황을 라이브로 처리 → AI 필연 시연.
3. **[스트레치] 사진 입력 = 멀티모달 비전 API 판독**: 별도 OCR 엔진 설치 X. 촬영 사진을 비전 가능 LLM에 직접 줘서 "읽기+이해"를 한 번에. (키트 web/lib/llm.ts는 텍스트 전용 → 이미지 호출 경로 1개 추가 필요, Stage 04에서 확정.) 사진 판독 자체가 규칙·검색 불가 = AI 필연 논거 강화. **단 라이브 외부 API+즉석 사진 품질 변수 → 샘플 이미지 폴백 + 붙여넣기 유지.**
4. **[스트레치] 전입신고 시나리오**: "같은 엔진이 일상 민원도" — 상황설명 입력(기한 14일).
5. **[스트레치] 위치/지도**: 관할 위치 표시. **카카오 무키 폴백 유지.** 관할=주소/사건 기준(거리 아님).
6. **[언급만] 음성·다국어**: 고령자·외국인 확장 스토리.

## 안전장치 (레드팀 반영 — 범위 유지하되 생존성 확보)
- OCR·지도는 폴백 없이 데모 메인 경로에 못 박지 않음.
- AI 필연은 ①임의 문서 해독 ②시민언어→행정개념 2가지로만 발표(과장 금지).
- 관할 확신 없으면 "정확한 소관은 여기서 확인" 링크로 회피(신뢰 원칙과 일관).
- 정부 흡수 반격 답: "받는 방향 입구는 정부포털 신청메뉴 구조와 충돌 + 법원·보험 등 비-정부24 문서도 처리"가 구조적 해자.

## 엔진 = AI only (사용자 확정 2026-06-20)
- 문서 읽기·이해·쉬운 말·첫 걸음·관할 = **전부 멀티모달 비전 + LLM**. 규칙 테이블·fixture 하드코딩으로 만든 '가짜 바닥'·키없음 규칙치환 **안 만듦**(이중 빌드 제거).
- **단 출처 근거(RAG)는 유지** — AI가 빈손 생성이 아니라 공식 출처 문서를 근거로 답·인용하고, 근거 없으면 "공식 확인 필요"로 회피. 이건 규칙 폴백이 아니라 신뢰 레이어(환각 차단=차별점, 프랑스 Albert ↔ NYC 반면교사).
- 출처용 권위 자료(지급명령·전입신고 등 공식 안내/법령 스니펫)는 AI가 인용할 **근거 소스**로만 준비(전수 강박 X, 데모 시나리오 커버 우선).
- 외부 의존은 OPENAI_API_KEY 한 곳. **데모 보험 = 코드 폴백 아님, 키 사전준비 + 시연 녹화 1본**(라이브 키/네트워크 실패는 의식적으로 받는 베팅).
