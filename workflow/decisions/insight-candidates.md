# Stage 02 — Insight 후보 & 방향 (승인 전 초안)

> 발산 2라운드(사용자 주고받기) → 수렴. 구조: Common Assumption → Observed Reality → Tension → Reframing.
> 평가축: 새로움 / 설득력 / 근거가능성 / AI연결성 / 시연가능성 / 4시간 구현 / 위험한 전제.
> 입력: research/integrated-findings.md, judge-review.md, overseas.md. **확정 아님 — 레드팀·사용자 승인 전.**

---

## Insight 후보 (6개)

### 후보 ① 신청주의의 역설 — "정보 부족"이 아니라 "확신 부족"
- Common: 혜택을 못 받는 건 정보 부족 → 더 알려주면 된다.
- Reality: 정보는 이미 넘침(복지로·정부24, 안내 128종). 그래도 안 받음.
- Tension: 정보량↑ ≠ 수급↑. "나는 안 될 것 같아"라는 자기검열로 시도조차 안 함.
- **Reframing: 미수급은 정보 부족이 아니라 "자격이 되는지에 대한 확신 부족" 문제.**
- 평가: 새로움 高 / 설득력 高 / 근거 中(자기검열 수치 unverified) / AI연결 中 / 시연 中 / 위험전제=감정 서사 과의존.

### 후보 ② 로그인 장벽의 아이러니 — 정확하려다 도달을 막는다
- Common: 정확한 맞춤엔 본인인증·정보연계가 필요.
- Reality: 인증 벽에서 가장 도움 필요한 사람이 먼저 이탈.
- Tension: 정확성을 위한 인증이 도달을 막음.
- **Reframing: 맞춤 복지의 첫 관문은 데이터 연계가 아니라 "로그인 없이 30초에 감 잡게 하는 것".**
- 평가: 새로움 中 / 설득력 高 / 근거 高(복지로·보조금24 로그인 전제 검증) / AI연결 低 / 시연 高 / 4시간 高. **→ 차별화·실용 방패. 보조 채택.**

### 후보 ③ 목록이 아니라 "이유" — 신뢰는 근거에서 온다 ★사용자 선택
- Common: 혜택 목록을 보여주면 알아서 판단한다.
- Reality: 목록만으론 "이게 왜 나한테?"를 몰라 안 믿고 안 신청.
- Tension: 목록 제공 ≠ 행동.
- **Reframing: 복지 안내의 핵심은 "무엇을 받나"가 아니라 "왜 당신이 해당되는가"의 근거.**
- 평가: 새로움 高 / 설득력 高 / 근거 高 / AI연결 高(근거 설명 생성) / 시연 高 / 4시간 中. **→ 척추.**

### 후보 ④ 찾기와 신청의 단절 — 진짜 목표는 "받는 것"
- Common: 혜택을 찾아주면 역할 끝.
- Reality: 찾은 뒤 "어디서 어떻게"를 몰라 흐지부지.
- Tension: 발견 ≠ 행동.
- **Reframing: 사용자의 일은 "아는 것"이 아니라 "실제로 받는 것" → 신청 동선까지가 서비스.**
- 평가: 새로움 中 / 설득력 高 / 근거 高 / AI연결 低 / 시연 中 / 4시간 高(외부 링크). **→ 보조 채택.**

### 후보 ⑤ AI의 자리는 "판정"이 아니라 "통역" ★사용자 선택
- Common: 복지에 AI를 쓰면 자격을 똑똑하게 판정해준다.
- Reality: AI 판정 = 환각 → 거짓 안내 → 신뢰 붕괴(금지).
- Tension: 판정에 쓰면 위험, 안 쓰면 'AI 프로젝트' 아님.
- **Reframing: AI의 자리는 판정이 아니라 "막연한 말을 구조화(입력) + 차가운 규칙 결과를 사람 말로 풀어주는(설명)" 통역사.**
- 평가: 새로움 高 / 설득력 高 / 근거 高 / AI연결 高(본질) / 시연 高 / 4시간 中 / 위험전제=설명 생성이 데이터 값만 쓰도록 제약 필요. **→ 척추(AI Leverage 정당화).**

### 후보 ⑥ 검색이 아니라 발견 — 모르는 건 검색도 못 한다
- Common: 사용자는 원하는 혜택을 검색한다.
- Reality: 뭘 받을 수 있는지조차 몰라 검색 키워드가 없음.
- Tension: 검색은 "아는 것"만 찾게 함.
- **Reframing: 필요한 건 검색창이 아니라 "안 물어봐도 먼저 짚어주는 발견".**
- 평가: 새로움 中 / 설득력 中 / 근거 中 / AI연결 中 / 시연 中. **→ 프레이밍 흡수(헤드라인 아님).**

## 좁힌 최종 방향 (2개)

### 방향 A (추천·사용자 확정) — "무엇을이 아니라 왜 당신이"
- **척추 Insight:** ③ + ⑤. 복지 안내의 핵심은 "왜 당신이 해당되는가"이고, AI는 그 "왜"를 통역하는 자리에 있다.
- **킥(차별점):** 자연어로 가족 상황 입력 → AI가 구조화 → 규칙 엔진이 대표 혜택과 결정론적 매칭 → **혜택마다 "왜 해당되나요?" 근거 배지 + AI 생성 한 줄 설명.**
- **AI Leverage (독립 평가):** 2곳에 본질적. (1) 자연어 입력 구조화, (2) fixture 규칙값 기반 근거 설명 생성. **자격 판정은 규칙 엔진**(LLM 금지) → 거짓 안내 차단.
- **보조 요소:** ② 무로그인·무저장(실용+개인정보 안전+차별화 방패), ④ 신청 동선(카드에 신청 채널·한 줄 신청법).
- **Wow Moment:** "AI가 내 말을 알아듣고, *왜* 되는지 설명까지 해주는" 순간.
- **기억에 남을 한 문장:** "혜택 목록을 주는 게 아니라, 당신이 *왜* 받을 수 있는지를 설명해드립니다."
- **데모 시연성:** 高 (클라이언트 규칙 + LLM 2지점, 무로그인·무저장 → 서버/DB 불필요).

### 방향 B (대안) — "로그인 없이 30초, 숨은 혜택 발견"
- 척추 Insight: ② + ⑥. 무로그인 즉시 발견이 핵심.
- 킥: 입력 3개·30초 → "해당될 수 있는 혜택 N건 발견"(전수 비교 아님, 발견 프레이밍).
- AI Leverage: 약함 — 발견 자체엔 AI 불필요(규칙만). ⑤를 안 넣으면 AI 약점 재발.
- 판정: 방향 A가 AI 정당성·신뢰·차별화에서 우위. B는 A의 보조 요소로 흡수.

## 데이터/정직성 가드 (전 방향 공통)
- 자동 전수 매칭 불가(복지로 CSV에 자격 필드 없음) → **큐레이션 fixture 8~15개** + 화면에 "예시 데이터(전체 아님)" 상시 표기.
- "놓친 N건"(전수 암시) 금지 → "해당될 수 있는 N건 발견".
- "참고용 추정, 확정은 기관" 면책 필수. 자격 판정 LLM 금지.

## 레드팀 반영 — 방향 A 확정 보강 (조건부 통과 → 4개 조건 해소)
레드팀(깐깐한 심사위원) 검토 결과 ③은 단단·② 무로그인은 진짜 무기로 평가. 단 아래 4개를 설계에 박는다:

1. **AI 본질성 (환각 방지 + "장식 아님" 증명):** 설명 생성 LLM은 **사실을 만들지 않는다.** 규칙엔진이 `{혜택명·충족조건·금액}`을 확정 → LLM은 그 **확정 텍스트를 사람 말로 다듬는(rewrite) 역할만.** 출력에 fixture 외 혜택명/숫자가 섞이면 **폴백 템플릿으로 자동 대체**(검증 단계). 시연 입력 1~2개는 설명 **사전 캐시**(라이브 LLM 죽어도 데모 생존). → 이 설계가 곧 ⑤("AI에 사실 생성 권한을 안 준다")를 인사이트로 정당화.
2. **킥의 시각화:** "왜 당신이"를 텍스트 문단이 아니라 **시각적 충족 체크리스트(✓ 소득 충족 ✓ 자녀수 충족) + "왜?" 배지**로. 0.5초에 꽂히게. 가능하면 "복지로(목록만) vs 우리(목록+왜 배지)" 대비 컷.
3. **차별화 진술 = ③ + ② 묶음:** "복지로는 *로그인·정보연계 동의* 후 '무엇을'만 준다. 우리는 *로그인 0회*로 '왜 당신인지'까지." ②를 보조가 아니라 차별화 문장의 동등한 절반으로.
4. **주장 수위 정직화:** "가족 맞춤" → **"예시 가족 시나리오"**. "예시 데이터(8건)" 라벨을 **데모 화면에 상시 노출**(문서가 아니라 화면).

**한 문장 재작성(대비 구조):** 후보 — (a) "정부는 '받을 수 있다'고만 합니다. 우리는 '왜 당신인지'를 보여줍니다." / (b) "복지로는 로그인해야 '목록'을 줍니다. 우리는 로그인 없이 '이유'를 줍니다." → 최종은 **사용자 선택.**
