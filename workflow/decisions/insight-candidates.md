# Stage 02 — 인사이트 후보 (발산→수렴, 승인 전 초안)

> 사용자와의 발산(여러 라운드: 정부24 중복 도전 → C로 선회 → 입력 방식 → 데모 문서 → 앱/위치)을 인사이트로 구조화.
> 형식: Common Assumption → Observed Reality → Tension → Reframing + AI Leverage + 킥. **승인 전 — 확정 아님.**

---

## 인사이트 후보 (≥5)

### I1. "행정 접근성 = 정보 부족" 이라는 착각 ★메인
- **Common Assumption:** 행정이 어려운 건 정보가 흩어져 있어서다 → 통합 포털을 만들면 된다.
- **Observed Reality:** 정부24가 이미 통합했고 AI 챗봇까지 붙였는데, 사람들은 여전히 *겁부터 내고* 인터넷을 헤맨다. 진짜 막히는 순간은 "받은 무서운 문서/처음 닥친 일" 앞의 **막막함·공포**다.
- **Tension:** 정보는 충분한데도 "첫 걸음"을 못 뗀다. 통합이 더 돼도 이 공포는 안 풀린다.
- **Reframing:** 행정 접근성은 *정보 부족* 문제가 아니라 **"첫 걸음 앞의 공포·막막함"** 문제다. 필요한 건 더 많은 정보가 아니라 **겁을 없애고 첫 걸음을 떼게 하는 온램프(on-ramp)**.
- **AI Leverage:** 시민이 받은 임의의 문서/모호한 상황을 → 의미 파악 + 쉬운 말 + *그 문서 고유의 다음 한 걸음*으로 변환. 문서마다 달라 검색·FAQ·규칙으로 불가 → 본질적으로 LLM.
- **킥:** "받은 문서를 찍거나 상황을 적으면 → 뜻 + 기한 + 첫 걸음 + 어디로 + 출처". 정부24가 안 하는 **'받는 방향' 진입점**.

### I2. 행정의 입구는 "신청"이 아니라 "내가 받은 문서"
- **Common Assumption:** 사용자는 "무슨 서비스를 신청할지" 알고 들어온다(정부24의 설계 전제).
- **Observed Reality:** 진짜 막히는 사람은 "내가 받은 이 종이가 뭔지"조차 모른다 — 신청이 아니라 **해독**이 출발점.
- **Tension:** 모든 행정 UI가 "신청(미는)" 기준인데, 시민의 막힘은 "수신(받는)"에서 시작된다.
- **Reframing:** 행정 서비스의 입구는 "신청 메뉴"가 아니라 **"내가 받은 문서/문자"** 여야 한다.
- **AI Leverage / 킥:** 임의 문서 OCR·이해 → 해독. (I1의 진입 축. 단독 인사이트로도 성립.)

### I3. 행정 AI의 가치는 "똑똑함"이 아니라 "정직함"
- **Common Assumption:** AI 행정 도우미의 가치는 *정확하고 똑똑한 답*이다.
- **Observed Reality:** 행정은 틀리면 피해가 크고(지급명령 2주 놓치면 확정), 사용자는 "이게 맞는지" 못 믿는다. 뉴욕시 MyCity 챗봇은 똑똑한 척하다 위법 조언으로 폐지됐다.
- **Tension:** 더 그럴듯한 답일수록 출처가 없으면 더 위험하다.
- **Reframing:** 행정 AI의 가치는 똑똑함이 아니라 **"출처를 보여주고, 모르면 모른다고 하는 정직함"**이다.
- **AI Leverage / 킥:** 출처 강제 RAG + '모름→공식 링크' 폴백(프랑스 Albert ↔ NYC 반면교사). 신뢰=차별 핵심.

### I4. "끝까지 다 알려주는" 친절이 사람을 더 겁먹게 한다
- **Common Assumption:** 행정 안내는 전체 절차를 끝까지 보여줘야 친절하다.
- **Observed Reality:** 전 과정을 펼치면 *끝이 안 보여 더 겁난다*. 사람을 멈추게 하는 건 "지금 첫 걸음" 하나다.
- **Tension:** 완전한 안내 ↔ 압도. 정보량과 행동 가능성이 반비례할 수 있다.
- **Reframing:** 친절함은 "전부 다"가 아니라 **"지금 당장 할 한 걸음"**을 주는 것. 나머지(실제 처리)는 정부24로 넘긴다 → 경쟁이 아니라 온램프.
- **킥:** 첫 걸음 크게 + 전체경로 접힌 미리보기(끝이 보이니 안심) + 정부24 핸드오프(경계 명확).

### I5. 접근성은 "쉬운 화면"이 아니라 "시민의 언어로 통역"
- **Common Assumption:** 디지털 약자(고령자·외국인)에겐 더 쉬운 UI가 필요하다.
- **Observed Reality:** 화면을 쉽게 해도 *용어·맥락*을 모르면 못 쓴다. 외국인은 행정복지센터에 가서도 '행정 단어'를 못 알아듣는다.
- **Tension:** 쉬운 화면 ≠ 알아듣는 언어.
- **Reframing:** 접근성은 "쉬운 화면"이 아니라 **"시민의 언어로 양방향 통역"**(공문→쉬운 말, 상황→행정 개념) 문제다. 음성·다국어로 확장 가능.
- **AI Leverage / 킥:** 양방향 통역. (약자 타깃 확장 스토리.)

### I6. 위치의 가치는 "가까움"이 아니라 "정확한 관할"
- **Common Assumption:** 가까운 기관을 알려주면 편하다.
- **Observed Reality:** 행정 관할은 *거리*가 아니라 *주소·사건* 기준 → "가장 가까운 곳"이 틀린 답일 수 있다(전입신고=새 주소지 관할, 이의신청=해당 법원).
- **Tension:** 편의(최근접) ↔ 정확(관할).
- **Reframing:** 위치의 가치는 "가까움"이 아니라 **"정확한 관할을 짚고, 그 위치를 지도로 보여주는 것"**.
- **킥:** 주소/문서 기반 관할 매핑(regions.ts) + 지도. "가까운 거 아무거나" 금지(틀린 안내 방지).

## 수렴 — 최종 방향 후보 (2~3개)

### ▶ 방향 A (추천): "겁먹지 않게, 첫 걸음을 떼주는 행정 온램프"
- **결합:** I1(공포 리프레이밍) + I2(받는 방향 진입) + I3(정직한 신뢰) + I4(첫 걸음만) + I6(정확한 관할).
- **한 줄:** 무서운/처음인 행정 문서·상황을 → 쉬운 말 + 기한 + **당장 첫 걸음** + 어디로(관할·지도) + **출처**로 바꿔주고, 실제 처리는 정부24로 넘기는 모바일(PWA) 도구.
- **AI Leverage:** 임의 문서/상황 → 해독 + 문서 고유의 첫 걸음 생성(검색 불가, LLM 필연). 출처 강제 RAG.
- **데모(킥):** ① 지급명령(받은 무서운 문서, 사진/붙여넣기, 기한 2주) ② 전입신고(처음 닥친 일상, 상황 설명, 기한 14일). 같은 엔진, 입력 2방식·감정 2결.
- **마지막 문장 후보:** "정부24는 '신청 버튼'을 줍니다. 우리는 그 버튼이 어디 있는지조차 모르는 사람을 거기까지 데려다줍니다."

### ▶ 방향 B (대안): "쉬운 말 통역기 중심 (약자 접근성)"
- **중심:** I5. 공문→쉬운 말 + 다국어·음성으로 고령자·외국인 정조준.
- **약점(리서치 경고):** 단독이면 "요약 봇/범용 LLM"과 차별 약함. 신뢰·행동(I3·I4) 없이는 격하 위험.

### ▶ 방향 C (대안): "신뢰 레이어를 전면에 (정직한 행정 AI)"
- **중심:** I3. "출처 + 모름 인정"을 메인 메시지로.
- **약점:** 신뢰는 강력한 *축*이지만, 단독 제품이라기보다 A의 핵심 기능. 메인 메시지로 세우면 데모가 추상적이 될 위험.

## 레드팀(적대적 심사위원) 결과 — 승인 전 1회 수행

> codex 미작동 → 클로드 리뷰어 폴백. 종합: **PASS-with-warnings**. 전략·프레이밍·피치는 강함. 범위·데모설계는 보완 필요.

**치명타 2개:**
1. 범위 팽창(2데모+OCR+지도+RAG) → 4시간에 전부 절반만 동작 → 라이브 실패 시 신뢰(I3) 자가당착.
2. 선택 데모 2개 모두 정적 규칙으로 하드코딩 가능 → "AI 어디 썼냐" 방어 불가. 즉석·미지 입력이 없으면 AI 필연이 *안 보임*.

**정직성 교정(중요):** AI 필연은 **I1·I2(임의 문서 해독 / 시민언어→행정개념)뿐.** I3(출처RAG=검색+인용)·I4(첫걸음=규칙테이블)·I6(관할=룩업)은 AI 지능 아님. 발표에서 "검색으로 대체 불가능한 2가지"로 한정.

**합의된 보완(핵심/스트레치 층 분리):**
- 핵심(끝까지 동작): **지급명령 1시나리오** + 붙여넣기/타이핑 + **즉석 입력 1개**(AI 필연 시연) + 출처 인용 + 첫걸음 + 관할(텍스트) + 환각 폴백.
- 스트레치(보너스, 깨져도 데모 안 죽음): 사진 OCR(샘플 폴백), 전입신고 2번째 시나리오, 지도/카카오(키 있을 때), 음성·다국어.
- 관할 정확도: 확신 없으면 "정확한 소관은 여기서 확인" 링크로 회피(I3와 일관).
- 정부 흡수 반격 답: "받는 방향 입구는 정부포털 '신청 메뉴' 구조와 충돌 + 법원·보험 등 비-정부24 문서도 다룸"이 구조적 해자.
